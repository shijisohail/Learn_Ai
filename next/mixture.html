01 — Concept

What is Mixture-of-Agents?

Not to be confused with Mixture-of-Experts (MoE) — which is an internal model architecture. Mixture-of-Agents is an external routing pattern: one request → router → appropriate model → response.

⚡
The key insight: A simple factual question doesn't need Claude Opus. A haiku doesn't need a code-specialist model. Using the wrong model is either wasting money (over-powered) or degrading quality (under-powered). A router solves this.

02 — Strategies

Model Routing Strategies

Four approaches to routing, ranging from dead-simple to sophisticated. Start simple and add complexity only when you have data showing it's needed.

📏
Rule-based routing
Simple keyword or pattern matching. If "code" or "debug" in query → code model. If query length < 100 tokens → cheap model. If "financial" or "legal" → careful model with disclaimers. Fast, zero latency overhead, fully predictable.
Easiest — start here
🏷️
Classifier-based routing
Train a small text classifier (BERT-based, ~100MB) that predicts the optimal model tier from the input. Latency: ~5ms. Requires a labeled routing dataset — collect from production traffic by logging which model was used and the quality outcome.
Better — needs training data
🧠
LLM-based routing
Use a fast cheap model (Haiku) to classify the query, then route to the appropriate model. The classifier prompt gives it a rubric for complexity assessment. Latency overhead: ~300ms. Highly accurate but adds one LLM call per request.
Smart — best accuracy
💰
Cost-aware escalation
Try the cheapest model first. If confidence is low or quality check fails, escalate to the next tier. "Try Haiku, if response quality score < 3.5, retry with Sonnet." Maximizes cost savings while maintaining quality SLA. Adds latency on escalation.
Advanced — best cost/quality tradeoff
03 — Implementation

Building a Router

A FastAPI service that classifies incoming requests and routes to the appropriate model. Supports the LLM-based classification strategy with cost-aware fallback.

PYTHON — FastAPI Intelligent Router
from fastapi import FastAPI, HTTPException from pydantic import BaseModel import anthropic, asyncio, json, time from typing import Literal from dataclasses import dataclass, field app = FastAPI() client = anthropic.AsyncAnthropic() class RouteRequest(BaseModel): prompt: str max_tokens: int = 1024 require_tier: Literal["auto", "haiku", "sonnet", "opus"] = "auto" MODEL_TIERS = { "haiku": {"model": "claude-haiku-3-5", "cost_per_1k": 0.0008}, "sonnet": {"model": "claude-sonnet-4-5", "cost_per_1k": 0.003}, "opus": {"model": "claude-opus-4-0", "cost_per_1k": 0.015}, } CLASSIFIER_PROMPT = """Classify the complexity of this user request for model routing. Request: {prompt} Rules: - haiku: simple factual Q&A, greetings, yes/no, <100 word summaries, single-step tasks - sonnet: code review, analysis, multi-step reasoning, 100-1000 word generation, debugging - opus: complex strategy, high-stakes decisions, long documents (>2000 tokens), difficult math Return ONLY valid JSON: {{"tier": "haiku"|"sonnet"|"opus", "confidence": 0.0-1.0, "reason": "brief explanation"}}""" async def classify_request(prompt: str) -> tuple[str, float]: """Use Haiku to classify the request tier. Returns (tier, confidence).""" response = await client.messages.create( model="claude-haiku-3-5", max_tokens=128, messages=[{"role": "user", "content": CLASSIFIER_PROMPT.format(prompt=prompt[:500])}], ) data = json.loads(response.content[0].text) return data["tier"], data["confidence"] async def call_model(tier: str, prompt: str, max_tokens: int) -> dict: """Call the specified model tier.""" model_info = MODEL_TIERS[tier] start = time.monotonic() response = await client.messages.create( model=model_info["model"], max_tokens=max_tokens, messages=[{"role": "user", "content": prompt}], ) latency_ms = int((time.monotonic() - start) * 1000) tokens_used = response.usage.input_tokens + response.usage.output_tokens return { "content": response.content[0].text, "tier_used": tier, "model": model_info["model"], "tokens_used": tokens_used, "cost_usd": tokens_used / 1000 * model_info["cost_per_1k"], "latency_ms": latency_ms, } @app.post("/v1/route") async def route_request(req: RouteRequest) -> dict: if req.require_tier != "auto": # Explicit tier override — skip classifier return await call_model(req.require_tier, req.prompt, req.max_tokens) # Run classifier concurrently with first attempt at Haiku # (optimistic: if haiku is enough, we're already done) tier, confidence = await classify_request(req.prompt) if confidence < 0.7: # Low confidence → escalate one tier escalation = {"haiku": "sonnet", "sonnet": "opus", "opus": "opus"} tier = escalation[tier] return await call_model(tier, req.prompt, req.max_tokens)
04 — Ensembles

Model Ensembles

Run the same query on multiple models simultaneously, then aggregate. Higher cost, but significantly better accuracy for high-stakes decisions where correctness matters more than speed.

🎯
When ensembles are worth the cost: Medical/legal advice, high-stakes factual questions, security-critical decisions. For most applications, a well-routed single model is better value. Use ensembles for <5% of your traffic.
PYTHON — Ensemble with Majority Vote
import asyncio from collections import Counter async def query_model(client, model: str, prompt: str) -> str: response = await client.messages.create( model=model, max_tokens=512, messages=[{"role": "user", "content": prompt}], ) return response.content[0].text async def ensemble_vote( prompt: str, models: list[str] = ["claude-haiku-3-5", "claude-sonnet-4-5", "claude-sonnet-4-5"], ) -> str: """Run prompt on multiple models, return majority answer. Best for classification/structured tasks where answers are comparable.""" client = anthropic.AsyncAnthropic() # Query all models in parallel responses = await asyncio.gather(*[ query_model(client, model, prompt) for model in models ]) # For structured output (e.g., classification), vote vote_counts = Counter(responses) winner, count = vote_counts.most_common(1)[0] print(f"Ensemble: {len(models)} models voted, winner got {count}/{len(models)} votes") return winner async def ensemble_judge( prompt: str, models: list[str] = ["claude-haiku-3-5", "claude-sonnet-4-5", "claude-opus-4-0"], ) -> str: """Run on multiple models, use a judge to pick the best response. Better for open-ended generation where voting doesn't apply.""" client = anthropic.AsyncAnthropic() responses = await asyncio.gather(*[ query_model(client, model, prompt) for model in models ]) # Judge prompt — present all responses, ask to pick best judge_prompt = f"""You are evaluating multiple AI responses to this prompt: {prompt} Responses: {chr(10).join(f'Option {i+1}: {r}' for i, r in enumerate(responses))} Which option is most accurate, helpful, and complete? Reply with ONLY "Option N" where N is the number.""" judgment = await query_model(client, "claude-sonnet-4-5", judge_prompt) try: idx = int(judgment.split("Option ")[1].strip()) - 1 return responses[idx] except (IndexError, ValueError): return responses[0] # fallback to first
05 — Production

Production Architecture

Running a router in production adds operational complexity. Rate limiting, cost tracking, fallback chains, and latency budgets are non-negotiable before going live.

PYTHON — Production Router with Cost Tracking
import asyncio, time from collections import defaultdict from dataclasses import dataclass, field from typing import Optional @dataclass class RouterMetrics: # Track per-tier usage for cost attribution requests_per_tier: dict[str, int] = field(default_factory=lambda: defaultdict(int)) cost_per_tier: dict[str, float] = field(default_factory=lambda: defaultdict(float)) latency_per_tier: dict[str, list] = field(default_factory=lambda: defaultdict(list)) escalations: int = 0 errors: int = 0 def record(self, tier: str, cost: float, latency_ms: int): self.requests_per_tier[tier] += 1 self.cost_per_tier[tier] += cost self.latency_per_tier[tier].append(latency_ms) def report(self) -> dict: import statistics return { tier: { "requests": self.requests_per_tier[tier], "total_cost_usd": round(self.cost_per_tier[tier], 4), "avg_latency_ms": statistics.mean(self.latency_per_tier[tier]) if self.latency_per_tier[tier] else 0, } for tier in ["haiku", "sonnet", "opus"] } # Fallback chain: if primary tier fails, try next FALLBACK_CHAIN = {"haiku": "sonnet", "sonnet": "opus", "opus": None} async def call_with_fallback( client, tier: str, prompt: str, max_tokens: int, latency_budget_ms: int = 5000, metrics: Optional[RouterMetrics] = None, ) -> dict: current_tier = tier while current_tier is not None: start = time.monotonic() try: response = await asyncio.wait_for( client.messages.create( model=MODEL_TIERS[current_tier]["model"], max_tokens=max_tokens, messages=[{"role": "user", "content": prompt}], ), timeout=latency_budget_ms / 1000, ) latency_ms = int((time.monotonic() - start) * 1000) tokens = response.usage.input_tokens + response.usage.output_tokens cost = tokens / 1000 * MODEL_TIERS[current_tier]["cost_per_1k"] if metrics: metrics.record(current_tier, cost, latency_ms) return {"content": response.content[0].text, "tier": current_tier} except (asyncio.TimeoutError, Exception) as e: print(f"Tier {current_tier} failed: {e}. Trying fallback...") if metrics: metrics.errors += 1 current_tier = FALLBACK_CHAIN[current_tier] raise RuntimeError("All model tiers failed")
