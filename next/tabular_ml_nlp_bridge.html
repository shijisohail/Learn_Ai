Section 01

The Right Tool for the Signal

The single most expensive mistake is using a 70B LLM to sort rows of structured data you already have in SQL. Each tool has a domain where it dominates — learn the signals.

Classical ML

Gradient boosting / logistic regression

  • Input: Tabular, structured features
  • Labels: Many labeled examples (1k+)
  • Output: Calibrated probability
  • Latency: Microseconds–milliseconds
  • Cost: Near-zero per prediction
  • Debuggability: SHAP values, feature importance
✅ When you have: labeled structured data, need calibration, want low latency and cost, must audit decisions
LLM / RAG

Large language model + retrieval

  • Input: Unstructured text, docs, code
  • Labels: Few-shot or no labels needed
  • Output: Free-form reasoning + JSON
  • Latency: 200ms–5s
  • Cost: $0.001–0.05 per call
  • Debuggability: Chain-of-thought, evals
✅ When you have: unstructured input, reasoning tasks, few labels, need language understanding

Concept check

A 70B LLM is being used to classify support tickets into 8 categories. You have 50,000 labeled examples. What should you probably do instead?

With 50k labels and 8 classes, a fine-tuned text classifier or logistic regression on embeddings will outperform a large LLM and cost 100× less per prediction.

Section 03

Entity Resolution — the Hard Problem

"Acme Corp", "ACME CORP.", "Acme Corporation LLC" are the same business. ER is the foundational data problem in B2B data pipelines, CRM deduplication, and knowledge graph construction.

1
Blocking
Reduce candidate pairs from O(n²) to manageable
Cheap rules: same ZIP code AND same 3-letter name prefix → candidate pair. Without blocking, comparing every record to every other is quadratic and impossible at scale.
2
Scoring
Compute similarity features on candidate pairs
Levenshtein distance on name, Jaro–Winkler, exact match on EIN/DUNS, embedding cosine similarity of company description. Stack these as features into a binary classifier.
3
Clustering
Transitivity: A=B and B=C means A=C
Pairwise matches don't give you clusters. Use connected-components or hierarchical clustering with a threshold. Handle conflicts with a tiebreaker (highest-confidence edge wins).
4
LLM assist
Use LLM only for ambiguous edge cases
"Two hedge fund subsidiaries with the same name in different states" — this is where your LLM earns its cost. Routing only ambiguous pairs to an LLM keeps costs rational (1-5% of pairs).

Section 04

Lead Scoring — Practical Pipeline

A canonical tabular ML task. Features: firmographics + behaviour + intent signals. Target: P(convert within 90 days). The same principles apply to churn, upsell, and fraud detection.

from sklearn.pipeline import Pipeline
from sklearn.preprocessing import StandardScaler, OrdinalEncoder
from sklearn.compose import ColumnTransformer
from sklearn.ensemble import GradientBoostingClassifier
from sklearn.model_selection import cross_val_score
import pandas as pd, numpy as np

# Feature groups
NUM_FEATURES = ['employee_count', 'annual_revenue', 'page_views_30d',
                'email_opens_7d', 'trial_days_active']
CAT_FEATURES = ['industry', 'country', 'source_channel']

preprocessor = ColumnTransformer([
    ('num', StandardScaler(), NUM_FEATURES),
    ('cat', OrdinalEncoder(handle_unknown='use_encoded_value',
                              unknown_value=-1), CAT_FEATURES),
])

pipeline = Pipeline([
    ('pre', preprocessor),
    ('clf', GradientBoostingClassifier(
        n_estimators=400, max_depth=4, learning_rate=0.05,
        subsample=0.8, random_state=42,
    )),
])

# Class imbalance: use PR-AUC, not accuracy
scores = cross_val_score(pipeline, X_train, y_train, cv=5,
                         scoring='average_precision')
print(f"CV PR-AUC: {scores.mean():.3f} ± {scores.std():.3f}")
from sklearn.calibration import CalibratedClassifierCV, calibration_curve
import matplotlib.pyplot as plt

# Wrap the model in Platt scaling (logistic) or isotonic regression
calibrated = CalibratedClassifierCV(pipeline, cv=5, method='sigmoid')
calibrated.fit(X_train, y_train)

# Verify calibration plot (reliability diagram)
prob_true, prob_pred = calibration_curve(
    y_test, calibrated.predict_proba(X_test)[:, 1], n_bins=10
)

# A well-calibrated model: P(y=1|score=0.7) ≈ 70% of the time
# If it's not: score of 0.7 means nothing to your sales team

# Expected calibration error (ECE)
ece = np.mean(np.abs(prob_true - prob_pred))
print(f"ECE: {ece:.3f}")  # Good: < 0.05
import anthropic, json, numpy as np

# Step 1: Extract structured signals from unstructured email body
def extract_intent_signals(email_text: str) -> dict:
    client = anthropic.Anthropic()
    resp = client.messages.create(
        model="claude-3-5-haiku-20241022",  # cheap extraction model
        max_tokens=256,
        messages=[{"role": "user", "content": f"""
Extract signals from this sales email. Return JSON only.
Fields: urgency(high/med/low), buying_stage(aware/consider/decide),
mentions_competitor(bool), pain_point_mentioned(str or null).

Email: {email_text}
"""}]
    )
    return json.loads(resp.content[0].text)

# Step 2: Encode extracted signals as additional tabular features
def build_hybrid_features(row: dict, email_signals: dict) -> np.ndarray:
    urgency_map = {'low': 0, 'med': 1, 'high': 2}
    stage_map   = {'aware': 0, 'consider': 1, 'decide': 2}
    return np.concatenate([
        extract_tabular_features(row),             # firmographics
        [urgency_map[email_signals['urgency']],     # from LLM
         stage_map[email_signals['buying_stage']],
         int(email_signals['mentions_competitor'])],
    ])

💡

Why calibration matters for lead scoring

Your sales team will act on "70% likelihood to convert". If that probability is not calibrated — i.e., only 40% of leads scored 70 actually convert — you've trained a useless tool. Always check your reliability diagram before shipping a scoring model to a business team.

Concept check

You train a GBDT on 1000 labeled leads with class imbalance (95 no-convert, 5 convert). Which metric should you optimize?

Correct: with 95/5 imbalance, accuracy is trivially gamed. PR-AUC measures how well you rank positives above negatives — the right objective for a scoring model.
