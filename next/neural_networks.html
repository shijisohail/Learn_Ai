01 — Foundations

What is a Neural Network?

A neural network is a function approximator: a parameterized mathematical function that learns to map inputs to outputs by adjusting millions (or billions) of numbers called weights. Inspired loosely by biological neurons.

🔑
The key insight: Each connection between nodes has a weight (a single floating-point number). Training adjusts all those weights so the network's outputs match what we want. A GPT-4-scale model has ~1.8 trillion such weights.
Node / Neuron
A single unit that receives inputs, computes a weighted sum, applies an activation, and produces one output value.
Weight (w)
A learnable scalar on each connection. Analogous to synapse strength. This is what training changes.
Bias (b)
An extra learnable offset added before the activation. Lets the neuron fire even when all inputs are zero.
Layer
A group of neurons that all process the same input vector in parallel, producing an output vector.
02 — The Math

The Math of a Single Neuron

Every neuron in every layer of every LLM executes the same simple formula. Billions of times, in parallel.

output = activation( Σ(wᵢ × xᵢ) + b )
xᵢ
Input values from the previous layer (or raw input)
wᵢ
Weights — one per input connection. These are what get trained.
Σ(wᵢxᵢ)
Weighted sum — a dot product. In matrix form: x · W
b
Bias — a learnable constant offset. Shifts the activation threshold.
activation()
Non-linearity. Without this, stacking layers adds zero expressiveness.
output
A single number fed to the next layer (or the final prediction).

Activation Functions — The Non-Linear Glue

Without activation functions, any stack of layers collapses to a single linear transformation. Activations introduce non-linearity, enabling networks to learn complex patterns.

Sigmoid
σ(x) = 1 / (1 + e⁻ˣ)
Squashes any input to (0, 1). Classic choice for binary outputs. Prone to vanishing gradients in deep nets — activations saturate near 0 and 1, killing gradient flow during backprop.
LSTM gates · Binary classifiers
ReLU
ReLU(x) = max(0, x)
Computationally trivial. Creates sparse activations (many zeros), which improves efficiency. Dominant for hidden layers in CNNs and older transformer FFNs. Dead neuron problem: neurons stuck at 0 never recover.
Hidden layers · CNNs · ResNets
GELU
x · Φ(x)  ≈  x · sigmoid(1.702x)
Gaussian Error Linear Unit. Smoother than ReLU — has a slight negative dip. Stochastically gates inputs: each input is multiplied by the probability it would be kept by a Gaussian dropout. Used in every modern transformer FFN.
GPT · Claude · Llama · BERT
Softmax
softmax(xᵢ) = eˣⁱ / Σeˣʲ
Converts a vector of raw scores (logits) into a probability distribution that sums to 1.0. Used in the final output layer of every LLM: each token in the vocabulary gets a probability. The model samples the next token from this distribution.
LLM output layer · Attention scores
03 — Training

How Networks Learn: Gradient Descent

Training a neural network is iterative optimization. You run data through the network, measure how wrong it is, compute which direction to adjust each weight, and nudge them slightly. Repeat for billions of steps.

1
Forward Pass
Input data flows through every layer left-to-right. Each neuron computes its weighted sum + activation. The network produces a prediction (e.g. probability distribution over tokens).
2
Loss Function
Measures how wrong the prediction was. For language modeling: cross-entropy loss — measures the negative log-probability of the correct next token. Lower loss = better prediction.
loss = -log( p(correct_token) )
3
Backpropagation
Using the chain rule of calculus, compute how much each weight contributed to the loss. Gradients flow backward through the network: from output → hidden layers → input. Each weight gets a gradient: ∂loss/∂w.
∂loss/∂w₁ = (∂loss/∂output) × (∂output/∂h) × (∂h/∂w₁) ← chain rule
4
Weight Update
Nudge each weight in the direction that reduces loss. The learning rate controls step size. Adam (used for most LLMs) adds momentum and adaptive learning rates per weight.
w = w - lr × ∂loss/∂w
↺
Repeat
Run another batch of data. Repeat for billions of steps across the entire training corpus. For GPT-4: approximately 13 trillion tokens, taking months on thousands of GPUs.

Key Training Terms

Learning Rate (lr)
How big a step to take per gradient update. Too high → diverges. Too low → slow convergence. Typically 1e-4 to 3e-4 for LLMs.
Batch Size
How many training examples to process before one weight update. Larger batches = more stable gradients, but need more GPU memory.
Epoch
One full pass over the training dataset. LLMs are often trained for less than 1 epoch on truly massive datasets (no repetition).
AdamW Optimizer
Adam + weight decay. Default for transformer training. Maintains per-weight momentum estimates and adapts step sizes. Used for Claude, GPT, Llama.
04 — Transformers

Deep Networks & Transformers

LLMs are just very deep neural networks. Each transformer block is a specific, carefully designed combination of neural network components.

One Transformer Block (repeated N times)
↑ residual stream (input embedding + positional encoding)
LayerNorm
Normalizes the residual stream. Stabilizes training.
↓
Multi-Head Attention
Q, K, V projections — neural network layers (linear). Scaled dot-product attention. Each head attends to different parts of context.
→ Neural net layers: W_Q, W_K, W_V, W_O
↓ + residual connection
LayerNorm
Applied before the FFN.
↓
Feed-Forward Network (FFN)
A 2-layer MLP. Expands to 4× hidden dim, applies GELU, projects back. This is literally the MLP from section 8.
FFN(x) = GELU(x·W₁ + b₁)·W₂ + b₂
↓ + residual connection → next block
💡
Parameters = weights. When we say "Claude has 70B parameters" we mean 70 billion learned weights — the W matrices and bias vectors in every attention layer and FFN across all transformer blocks. Training discovers the specific values of these numbers from data.
PYTHON — Transformer FFN is Just a 2-Layer MLP
Python (PyTorch-style pseudocode)
class TransformerFFN: # The "feed-forward" sub-layer inside every transformer block # d_model = 4096 for Llama-7B, 12288 for GPT-3 # d_ff = 4 × d_model (typical — the "expansion factor") def __init__(self, d_model, d_ff): self.W1 = Linear(d_model, d_ff) # expand self.W2 = Linear(d_ff, d_model) # contract def forward(self, x): # x shape: (batch, seq_len, d_model) h = gelu(self.W1(x)) # (batch, seq_len, d_ff) — this IS a neural net layer return self.W2(h) # (batch, seq_len, d_model) — project back # Llama-7B has 32 transformer blocks. # Each block has one of these FFNs. # FFN params per block ≈ 2 × 4096 × 16384 = 134M weights. # Total FFN params ≈ 32 × 134M = 4.3B — majority of total 7B params.
05 — Depth

Why Depth Matters

Deeper networks learn hierarchical representations. Each layer builds abstractions on top of what the previous layer learned. This is why "more layers" ≠ just "bigger" — it means "more compositional."

Early Layers
Simple, Local Patterns
Text: token co-occurrence, syntax neighbors. Vision: edges, colors, textures. These are statistical regularities in the raw data.
Mid Layers
Compositional Patterns
Text: phrases, named entities, syntactic roles. Vision: eyes, wheels, shapes. Combinations of the simple patterns from earlier layers.
Deep Layers
Abstract, Semantic Concepts
Text: sentiment, logical relationships, world knowledge, reasoning chains. Vision: "cat", "car", "person". High-level semantic features used for predictions.
📐
Universal Approximation Theorem: A neural network with a single hidden layer of sufficient width can approximate any continuous function to arbitrary precision. But depth is exponentially more efficient at representing certain functions than width. Composition is the secret — depth ~ function composition, width ~ capacity at each step.
Depth ≈ Composition
Each layer applies a transformation on top of the previous. Analogy: writing a poem → choosing genre → choosing theme → choosing words. Deep processing through abstraction levels.
Width ≈ Capacity
More neurons per layer = more patterns can be stored at that level of abstraction. A wider net can memorize more, but doesn't automatically generalize better. Modern LLMs are both deep AND wide.
06 — Architectures

Modern Architecture Variants

The MLP is the core primitive. Different architectures add structure — spatial, temporal, sparse — to handle different domains and constraints.

Architecture Core Idea Strength Used In Status
Transformer
Dominant 		Attention + FFN (MLP). Every token attends to every other token. Long-range dependencies, parallelizable training. Scales extremely well. GPT, Claude, Llama, Gemini, BERT, T5 Standard for language
CNN
Vision 		Convolutional filters: shared weights slide over spatial positions. Translation invariance, very efficient for images. Weight sharing = far fewer parameters. ResNet, VGG, EfficientNet, DALL-E encoder Standard for vision (+ ViT emerging)
LSTM / GRU
Legacy 		Recurrent: processes sequence one token at a time, passes hidden state forward. Memory over sequences without attending to all positions. Low memory at inference. Pre-transformer NLP, speech recognition Largely replaced by transformers
Mamba / SSM
2023–2024 		State Space Models: linear recurrence with selective gating. O(n) vs O(n²). Linear time complexity in sequence length. Efficient for very long contexts (100K+ tokens). Mamba, Jamba, Zamba Promising — watch this space
MoE
Sparse 		N expert sub-networks per layer; router activates only K of N per token (sparse). Massive parameter count with sub-linear compute. Scale capacity without proportional cost. GPT-4 (likely), Mixtral 8×7B, Switch Transformer Production at scale
🔀
Attention is O(n²): The cost of computing attention between all token pairs grows as the square of sequence length. This is why 100K+ context windows are expensive, and why Mamba (linear-time) is interesting for ultra-long contexts. Hybrid architectures mixing attention and SSMs are emerging (Jamba).
07 — In Your Stack

Neural Networks in Your LLM Stack

Everything you built in the 7-day curriculum has a neural network underneath. Here's where they show up.

🔢
Embedding Model (Day 3)
The embedding model you called (text-embedding-3-small) is a transformer encoder — a neural network that maps text to a dense vector in ~1536 dimensions. The vector is the activations at the final layer. Two semantically similar sentences produce vectors close in this space because the network was trained to do exactly that (via contrastive loss).
👁️
Claude's Attention (Day 1)
When Claude reads your prompt, each transformer layer runs multi-head attention — a set of neural network layers (W_Q, W_K, W_V, W_O) plus an FFN. The "context window" is the sequence length n; the quadratic cost is why 200K-token windows are expensive.
🎛️
Fine-tuning
Fine-tuning updates the weights of a pre-trained neural network on your domain data — running backpropagation again, but starting from already-good weights instead of random. Full fine-tuning touches all weights. LoRA adds tiny trainable weight matrices (rank decomposition) to frozen layers: neural network adapters on top of a frozen neural network.
→ Fine-tuning Deep Dive
🔀
Model Routing (Day 6)
When you built routing logic to send simple queries to Claude Haiku and complex ones to Sonnet, you were mimicking what MoE transformers do internally. Production routing classifiers are often small neural networks themselves — trained to predict which model will answer a given query best.
🧬
Constitutional AI / RLHF
The reward model in RLHF is a neural network fine-tuned to predict human preference scores for outputs. It takes a (prompt, response) pair and outputs a scalar reward. During PPO training, this reward signal is used to adjust the policy (the LLM) via backpropagation. Two neural networks training each other.
08 — Hands-On

Forward Pass From Scratch in Python

A complete, runnable 2-layer MLP — the same structure as the FFN inside every transformer block. No libraries beyond NumPy required.

▶
Run this now: Copy the code below, paste into any Python REPL or Jupyter cell. It has zero dependencies beyond NumPy (standard in any Python environment). It implements the same forward pass that runs inside every LLM on every token.
PYTHON — Minimal Neural Net from Scratch
Python
import numpy as np # ── Activation Functions ──────────────────────────────────────────────────── def relu(x): return np.maximum(0, x) def gelu(x): # Tanh approximation — same formula used in GPT-2, Claude, Llama return x * 0.5 * (1 + np.tanh(np.sqrt(2 / np.pi) * (x + 0.044715 * x**3))) def softmax(x): # Subtract max for numerical stability (prevents overflow in exp) e = np.exp(x - x.max()) return e / e.sum() # ── 2-Layer MLP: the FFN inside every transformer block ──────────────────── class MLP: """2-layer neural network — same structure as the FFN in GPT/Claude/Llama""" def __init__(self, d_in, d_hidden, d_out): # Xavier (Glorot) initialization: scale by sqrt(2/fan_in) # Prevents vanishing/exploding activations at initialization self.W1 = np.random.randn(d_in, d_hidden) * np.sqrt(2 / d_in) self.b1 = np.zeros(d_hidden) self.W2 = np.random.randn(d_hidden, d_out) * np.sqrt(2 / d_hidden) self.b2 = np.zeros(d_out) def forward(self, x): # Layer 1: linear transform + GELU activation # Shape: (d_in,) → (d_hidden,) h = gelu(x @ self.W1 + self.b1) # Layer 2: linear transform + softmax (for classification/language) # Shape: (d_hidden,) → (d_out,) out = softmax(h @ self.W2 + self.b2) return out # ── A tiny language model: vocab=5, hidden=8 ─────────────────────────────── np.random.seed(42) model = MLP(d_in=5, d_hidden=8, d_out=5) vocab = ['the', 'cat', 'sat', 'on', 'mat'] # One-hot encode "cat" (index 1) x = np.zeros(5); x[1] = 1.0 # Forward pass: "given 'cat', what's the next token?" probs = model.forward(x) predicted = vocab[np.argmax(probs)] print(f"After 'cat', model predicts: '{predicted}'") print(f"Probabilities: {dict(zip(vocab, probs.round(3)))}") # → random weights, so output is random. Training would fix this. # ── Count parameters (like "7B params" in LLM specs) ─────────────────────── n_params = (model.W1.size + model.b1.size + model.W2.size + model.b2.size) print(f"\nModel parameters: {n_params}") print(f" W1: {model.W1.shape} ({model.W1.size} weights)") print(f" W2: {model.W2.shape} ({model.W2.size} weights)") # Scale this up: Llama-7B has 32 layers, d_model=4096, d_ff=16384 # FFN params per layer = 4096×16384 + 16384×4096 = 134M per layer # Plus attention, embeddings, etc. → ~7 billion total
🎯
Extend the exercise: Add a training loop. Compute cross-entropy loss: loss = -np.log(probs[correct_idx]). Then implement backprop manually: compute gradients by hand using chain rule and update W1, W2 with W -= lr * grad_W. Train for 1000 steps on a tiny dataset ("the cat sat on the mat"). Watch the loss drop.

Run the forward pass and swap activation functions
Copy the MLP code. Run it with GELU, then swap for ReLU and sigmoid. Print the hidden layer activations h for each. Notice how GELU produces small negatives where ReLU is exactly zero. Understand why this matters for gradient flow.

Implement backpropagation and train the tiny LM
Add a training loop to the MLP. Implement cross-entropy loss, compute gradients manually using chain rule (dL/dW2, dL/dW1), update weights. Train on the sequence "the cat sat on the mat" for 2000 steps. Log loss every 100 steps. It should converge near 0.
Medium — 2 hours
3
Scale up: count parameters like a real LLM
Create an MLP with d_in=768, d_hidden=3072, d_out=768 (BERT-base dimensions). Count total parameters. Then multiply by 12 (number of transformer blocks). Compare to the stated BERT-base parameter count of 110M. Understand where those 110M numbers live.
Easy — 20 min
4
Visualize activation function gradients
Plot the derivative (gradient) of sigmoid, ReLU, and GELU using matplotlib. Observe: sigmoid saturates near 0/1 (gradient dies). ReLU is exactly 0 for x < 0 (dead neurons). GELU is always non-zero (always has gradient). This explains why GELU trains better in deep networks.
Medium — 1 hour
