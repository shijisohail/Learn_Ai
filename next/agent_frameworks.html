02 — Framework Cards

The Five Frameworks

Each card shows the paradigm, key strengths and weaknesses, install command, and a minimal working code example.

LangGraph
LangChain Inc. · 2024
Graph / State Machine
Build stateful, fault-tolerant agent workflows as explicit directed graphs. You define nodes (Python functions), edges (transitions), and a shared state schema. LangGraph adds checkpointing, streaming, and HITL on top.
pip install langgraph
ParadigmStateGraph + nodes + edges
HITLBuilt-in interrupt()
CheckpointingBuilt-in (SQLite / Redis)
StreamingToken + node-level
Best for
  • Complex multi-step production agents
  • Long-running workflows needing persistence
  • Human approval gates in the loop
  • Teams already on Python/FastAPI
Avoid when
  • Prototyping a simple single-agent script
  • Your team dislikes graph mental models
  • Content pipelines where roles are cleaner
📖
You covered this on Day 4. See day4.html for the full deep-dive on StateGraph, conditional edges, checkpointing, and HITL patterns.
PYTHON
from langgraph.graph import StateGraph, END from typing import TypedDict, Annotated import operator class AgentState(TypedDict): messages: Annotated[list, operator.add] next_step: str def research_node(state: AgentState) -> AgentState: # call an LLM or tool here return {"messages": ["research result"], "next_step": "write"} def router(state: AgentState) -> str: return state["next_step"] # conditional edge logic graph = StateGraph(AgentState) graph.add_node("research", research_node) graph.add_node("write", write_node) graph.add_conditional_edges("research", router, {"write": "write", "end": END}) app = graph.compile(checkpointer=memory) # built-in persistence
CrewAI
CrewAI Inc. · 2024
Role-Based
Define a "crew" of agents each with a role, goal, and backstory. Assign tasks to agents. The crew handles orchestration — sequential or hierarchical.
pip install crewai crewai-tools
ParadigmAgents + Tasks + Crew
HITLManual callback
CheckpointingNo
Best for
  • Content / research pipelines
  • Fast prototyping with roles
  • Non-engineers reading agent config
Avoid when
  • You need dynamic branching logic
  • Long-running jobs needing recovery
PYTHON
from crewai import Agent, Task, Crew, Process researcher = Agent( role="Senior Research Analyst", goal="Uncover cutting-edge AI developments", backstory="You work at a leading tech think tank...", tools=[search_tool, web_scraper], verbose=True ) writer = Agent( role="Tech Content Strategist", goal="Craft compelling content on tech advances", backstory="You are a renowned content strategist...", ) research_task = Task( description="Investigate latest AI agent frameworks", agent=researcher, expected_output="Comprehensive report with key findings" ) crew = Crew( agents=[researcher, writer], tasks=[research_task, write_task], process=Process.sequential ) result = crew.kickoff()
AutoGen
Microsoft Research · 2023
Conversation-Based
Agents communicate through messages. Any agent can reply to any other. Built-in code execution sandbox — the agent writes Python, runs it, reads stdout, iterates. Ideal for research and code-gen workflows.
pip install pyautogen
ParadigmConversableAgent messages
HITLUserProxyAgent built-in
Code SandboxBuilt-in Docker/local
Best for
  • Code generation + execution loops
  • Debate / critique agent patterns
  • Research assistants and data analysis
Avoid when
  • You need deterministic graph structure
  • Strict production reliability requirements
PYTHON
from autogen import AssistantAgent, UserProxyAgent assistant = AssistantAgent( name="assistant", llm_config={"model": "claude-opus-4-6", "api_key": "..."} ) user_proxy = UserProxyAgent( name="user_proxy", human_input_mode="NEVER", # fully autonomous code_execution_config={"work_dir": "coding"} ) # agent writes code, executes it, fixes errors automatically user_proxy.initiate_chat( assistant, message="Write and test a Python function to parse JWT tokens" )
OpenAI Agents SDK
OpenAI · 2025
Lightweight Loop
Minimal, opinionated agent loop from OpenAI. Define an agent with instructions + tools. Runner.run() handles the loop. Handoffs route between agents. Works with any model provider despite the name.
pip install openai-agents
ParadigmAgent + Runner + handoffs
HITLManual callback
TracingBuilt-in
Best for
  • Quick single-agent prototypes
  • Triage / routing between specialists
  • Minimal boilerplate, fast iteration
Avoid when
  • Complex state that needs persistence
  • Fine-grained conditional workflow control
PYTHON
from agents import Agent, Runner, handoff billing_agent = Agent( name="Billing", instructions="Handle invoice and payment issues", tools=[lookup_invoice, issue_refund] ) triage_agent = Agent( name="Triage", instructions="Route customer issues to the right specialist", handoffs=[billing_agent, tech_support_agent] ) # async runner handles the tool-call loop automatically result = await Runner.run(triage_agent, "My invoice is wrong") print(result.final_output)
DSPy
Stanford NLP · 2023
Compiler / Optimizer
DSPy is not a workflow framework — it's a prompt compiler. You declare what you want (Signature), compose modules (ChainOfThought, ReAct, ProgramOfThought), and an optimizer automatically finds the best prompts and few-shot examples for your metric. Think: write the logic, let DSPy find the words.
pip install dspy-ai
ParadigmDeclarative LM programs
HITLN/A
CheckpointingN/A
Key conceptSignatures + Optimizers
Best for
  • Automatic prompt optimization with labeled data
  • RAG pipelines where prompt quality matters
  • Reducing prompt engineering toil over time
  • Composable LM programs with typed I/O
Avoid when
  • You have no labeled training examples
  • You need agent loops / tool calling orchestration
  • Latency-sensitive production path (optimizer runs offline)
⚠️
Common misconception: DSPy does not replace LangGraph or CrewAI. It replaces hand-tuned prompts inside your agent nodes. Use DSPy to optimize a single classification or extraction step, then call that optimized module from your LangGraph node.
PYTHON
import dspy from typing import Literal # 1. Declare signature — what goes in, what comes out class SentimentClassifier(dspy.Signature): """Classify sentiment of customer feedback""" feedback: str = dspy.InputField() sentiment: Literal["positive", "negative", "neutral"] = dspy.OutputField() reasoning: str = dspy.OutputField(desc="Why this sentiment?") # 2. Compose module (DSPy handles the prompt template) classify = dspy.ChainOfThought(SentimentClassifier) # 3. Run optimizer — finds best prompts + few-shot examples automatically optimizer = dspy.BootstrapFewShot(metric=accuracy_metric) optimized_classify = optimizer.compile(classify, trainset=labeled_examples) # 4. Use the optimized module in production (or inside a LangGraph node) result = optimized_classify(feedback="Shipping was way too slow, very disappointed") print(result.sentiment) # "negative" print(result.reasoning) # "Expresses frustration about delivery time"

04 — Decision Guide

Which Should I Use?

Walk through the questions in order. The first condition that matches is your answer.

1
Do you need the agent to pause, resume, or survive restarts?
Long-running jobs, human approval gates, multi-day workflows.
→ LangGraph. Checkpointing and interrupt() are built-in. No other framework in this list gives you this.
2
Is the task a content or research pipeline with natural agent roles?
Researcher → Writer → Editor → Publisher flows. Non-engineer team members will read the config.
→ CrewAI. Role-based abstraction maps cleanly to this pattern. Lowest boilerplate for this use-case.
3
Does the agent need to write, execute, and iterate on code?
Code generation, debugging loops, data analysis scripts, test writing.
→ AutoGen. The built-in code execution sandbox is the differentiator. Agents write Python, run it in Docker, read stderr, and fix errors — automatically.
4
Do you just need a working agent with minimal boilerplate — fast?
Prototype, internal tool, or a simple triage/routing agent.
→ OpenAI Agents SDK. Fewest lines to a running agent. Handoffs between specialists work out of the box.
5
Do you have labeled examples and want automatic prompt optimization?
You know what good output looks like; you're tired of hand-tuning prompts; accuracy matters more than latency.
→ DSPy. Compile your LM program against your metric. Use alongside any of the above frameworks — DSPy optimizes individual nodes, not the whole orchestration.
6
Already using LangGraph and it's working?
It handles your current requirements. The team knows it.
→ Stay with LangGraph. Framework switching has real cost. Prefer depth over breadth — invest in LangGraph's production features (streaming, multi-agent subgraphs, LangSmith observability) before evaluating alternatives.
✅
The practitioner's shortcut: For most Python/FastAPI backend engineers — start with LangGraph for anything production, OpenAI Agents SDK for quick prototypes, and add DSPy when you have labeled data and prompt quality is a bottleneck. Revisit CrewAI and AutoGen for their specific niches (content pipelines and code-gen, respectively).
