Section 01

Enterprise AI Security Requirements

Enterprise AI has four non-negotiable requirements. Get any of them wrong and you lose the deal — or worse, cause a data breach.

  • Multi-tenancy: same AI system, completely isolated data per organization — tenant A must never see tenant B's data
  • Authorization: users within a tenant have different roles and permissions — a field rep cannot cancel contracts
  • Audit trail: every AI-triggered action must be logged with full context — who, what, when, why, outcome
  • Compliance: GDPR data residency, HIPAA PHI restrictions, SOC2 access control evidence — AI must be provably compliant

Section 02

JWT Integration with Agent Context

Extract scopes and tenant from the JWT at the API boundary. Pass auth context through agent state — never in the LLM prompt, never in tool descriptions.

  • Extract scopes and tenant from JWT at API boundary: one place to validate, then propagate via typed state
  • Pass auth context through agent state (not in the prompt): the LLM cannot be trusted to enforce authorization decisions
  • Never expose auth tokens in LLM context: the model's context window could be leaked — tokens belong in application state only
  • Short-lived tokens + refresh pattern: for long-running agent sessions, refresh tokens before they expire mid-execution
PYTHON
from fastapi import Depends, HTTPException from fastapi.security import HTTPBearer import jwt from pydantic import BaseModel from typing import Optional class AuthContext(BaseModel): user_id: str tenant_id: str scopes: list[str] user_role: str # "admin" | "manager" | "field_rep" security = HTTPBearer() def verify_jwt(token: str = Depends(security)) -> AuthContext: try: payload = jwt.decode(token.credentials, JWT_SECRET, algorithms=["RS256"]) return AuthContext( user_id=payload["sub"], tenant_id=payload["tenant_id"], scopes=payload.get("scopes", []), user_role=payload.get("role", "field_rep") ) except jwt.ExpiredSignatureError: raise HTTPException(status_code=401, detail="Token expired") except jwt.InvalidTokenError: raise HTTPException(status_code=401, detail="Invalid token") @app.post("/chat") async def chat(request: ChatRequest, auth: AuthContext = Depends(verify_jwt)): # Auth context flows into agent state, never into the prompt state = AgentState( messages=[HumanMessage(content=request.message)], auth=auth, # passes through the graph tenant_id=auth.tenant_id ) result = await agent_graph.ainvoke(state) return {"response": result["messages"][-1].content}
Section 03

Per-Tenant Vector Isolation

Tenant isolation must be enforced at the database layer, not just the application layer. Both layers together make a cross-tenant leak practically impossible.

  • Pinecone namespaces: built-in tenant isolation at the index level — namespace = tenant_id, enforced by the DB
  • Qdrant payload filtering: mandatory tenant_id filter on every query — application layer adds it, DB enforces it
  • Defense in depth: application-layer + DB-layer enforcement — even a code bug in one layer is caught by the other
  • Testing tenant isolation: automated adversarial tests — query as tenant A, verify zero results from tenant B's namespace
PYTHON
from qdrant_client import QdrantClient from qdrant_client.models import Filter, FieldCondition, MatchValue class TenantAwareRetriever: def __init__(self, qdrant_client: QdrantClient, collection: str): self.client = qdrant_client self.collection = collection def retrieve( self, query_vector: list[float], auth: AuthContext, doc_type: Optional[str] = None, k: int = 5 ) -> list[dict]: """Retrieve with mandatory tenant isolation""" must_conditions = [ FieldCondition( key="tenant_id", match=MatchValue(value=auth.tenant_id) ) ] if doc_type: must_conditions.append( FieldCondition(key="doc_type", match=MatchValue(value=doc_type)) ) results = self.client.search( collection_name=self.collection, query_vector=query_vector, query_filter=Filter(must=must_conditions), limit=k, with_payload=True ) # Defense in depth: verify at application layer too for r in results: assert r.payload["tenant_id"] == auth.tenant_id, \ f"Tenant isolation violation: got {r.payload['tenant_id']}, expected {auth.tenant_id}" return [{"content": r.payload["text"], "score": r.score} for r in results]
Section 04

RBAC Enforcement in Tool Calls

Authorization must be enforced in the execution layer, not in the LLM prompt. A prompt-based restriction is bypassed by jailbreak. A decorator-based restriction is not.

  • Tool permissions: map scope → allowed tools — the LLM proposes, the decorator decides
  • Enforce at execution layer, not in the LLM prompt: the model is not a security boundary — code is
  • Structured error responses for unauthorized tool calls: return a typed error the agent can reason about, not a crash
  • Audit every tool call: who, what, when, outcome — unauthorized attempts are as important to log as successful ones
PYTHON
from functools import wraps from typing import Callable # Permission map: scope → tools TOOL_PERMISSIONS = { "visits:read": ["get_visit_details", "list_visits", "get_customer_info"], "visits:write": ["schedule_visit", "update_visit_status", "add_visit_note"], "orders:read": ["get_order_details", "list_orders"], "orders:write": ["update_order", "cancel_order"], "admin": ["*"] # all tools } def require_scope(scope: str): """Decorator: enforce scope before tool execution""" def decorator(fn: Callable): @wraps(fn) def wrapper(*args, auth: AuthContext = None, **kwargs): if auth is None: raise PermissionError("No auth context provided") allowed_tools = set() for user_scope in auth.scopes: allowed_tools.update(TOOL_PERMISSIONS.get(user_scope, [])) tool_name = fn.__name__ if "*" not in allowed_tools and tool_name not in allowed_tools: # Log unauthorized attempt audit_log.warning( f"UNAUTHORIZED: user={auth.user_id} tenant={auth.tenant_id} " f"tool={tool_name} scopes={auth.scopes}" ) raise PermissionError( f"Insufficient permissions: '{tool_name}' requires scope '{scope}'" ) # Log authorized tool call audit_log.info( f"TOOL_CALL: user={auth.user_id} tenant={auth.tenant_id} tool={tool_name}" ) return fn(*args, auth=auth, **kwargs) return wrapper return decorator @require_scope("orders:write") def cancel_order(order_id: str, reason: str, auth: AuthContext = None) -> dict: """Cancel an order — requires orders:write scope""" return backend.cancel_order( order_id=order_id, reason=reason, cancelled_by=auth.user_id, tenant_id=auth.tenant_id # always scope to tenant )
Section 05

Audit Logging & Compliance

Every AI action must be traceable. Immutable audit logs are the evidence that satisfies SOC2 auditors, GDPR investigations, and internal security reviews.

  • Audit log schema: who (user_id), what (action + payload hash), when (timestamp), why (AI reasoning excerpt), outcome (success/failure)
  • Append-only audit log: immutable storage — S3 Object Lock, WORM-compliant databases — deletion is not allowed
  • PII handling in logs: tokenize sensitive fields before logging — store a hash, not the raw PII; respect retention limits
  • Compliance mapping: GDPR right-to-erasure requires tokenizing user_id — hash the ID so you can delete without breaking log integrity
PYTHON
import json import time from dataclasses import dataclass, asdict from enum import Enum class AuditEventType(str, Enum): AI_QUERY = "ai_query" TOOL_CALL = "tool_call" APPROVAL_REQUESTED = "approval_requested" APPROVAL_GRANTED = "approval_granted" APPROVAL_DENIED = "approval_denied" TOOL_EXECUTED = "tool_executed" TOOL_FAILED = "tool_failed" @dataclass class AuditEvent: event_type: AuditEventType timestamp: float user_id: str tenant_id: str session_id: str # Action details action: str payload_hash: str # hash of payload, not raw PII # Outcome outcome: str # "success" | "failure" | "pending" error: Optional[str] = None # AI context model: Optional[str] = None prompt_version: Optional[str] = None def log_audit_event(event: AuditEvent): """Write to immutable append-only audit log""" log_entry = json.dumps(asdict(event)) # Write to: CloudWatch Logs, S3 Object Lock, or append-only DB audit_stream.write(log_entry + "\n")
