02 — Core Infrastructure

Core Infrastructure

FastAPI with JWT middleware, Qdrant with per-tenant namespace isolation, Redis for session cache, PostgreSQL for LangGraph's checkpointer — initialized at startup via FastAPI lifespan.

03 — LangGraph Orchestrator

LangGraph Copilot Orchestrator

The complete copilot graph from Day 29 patterns, hardened with tenant isolation in state, interrupt_before execution, and PostgreSQL checkpointing for durable multi-turn conversations.

  • Nodes: intent_classify → entity_collect → propose_action → execute_action — four nodes, one clean flow
  • Interrupt before execute: Graph always pauses at execute_action for human confirmation — cannot be bypassed
  • Tenant isolation: thread_id in checkpointer = f"{tenant_id}:{user_id}:{session_id}" — separate conversation state per user
  • Auth context in state: JWT claims injected at graph entry — tools check permissions from state, not from re-decoding tokens
🔑
Security architecture: The FastAPI layer decodes and validates the JWT. The LangGraph layer enforces permissions at tool-call time using the claims already in state. No double-decoding, no trust escalation between layers.
04 — Guardrails Integration

Guardrails Integration

Two layers of guardrails: input validation (prompt injection, scope enforcement) and output validation (PII detection, structure check). Combined with audit logging middleware.

  • Input guardrails: Detect prompt injection attempts, validate that user intent is within their permitted scope, block tenant data cross-contamination attempts
  • Output validation: Pydantic model validation via Instructor, PII regex scan before returning, structured output schema enforcement
  • Audit logging middleware: Every message logged with full context — user_id, tenant_id, intent, action taken, timestamp — append-only to Postgres
  • RBAC at tool level: Each LangGraph tool function checks state["auth"]["scopes"] before executing — auth is not just at the API boundary
✅
Defense in depth: Multiple security layers means a single failure doesn't compromise the system. Guardrails AI catches injection → RBAC catches scope violations → audit log catches anomalies after the fact. Assume breach; detect it.
