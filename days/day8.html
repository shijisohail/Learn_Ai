01 — Overview

What is LangChain?

LangChain is a composability framework for LLM applications. It standardises the interface for models, prompts, retrievers, and tools so you can wire them together with minimal boilerplate — and swap providers without rewriting your logic.

💡
LangChain vs LangGraph: Use LangChain for stateless pipelines where data flows in one direction (RAG, summarisation, extraction). Use LangGraph when you need cycles, conditional branching, or persistent agent state between steps.
Core Runnable Interface
.invoke()
Synchronous single call — returns the full response. Use for scripts and REPL exploration.
.stream()
Sync streaming — yields chunks as they arrive. Use for CLI tools and SSE backends.
.batch()
Parallel batch — runs N inputs concurrently with a thread pool. Use for bulk processing.
.ainvoke()
Async invoke — awaitable single call. Use inside FastAPI handlers and async services.

02 — LCEL

LangChain Expression Language

LCEL uses the pipe operator | to compose Runnables into a RunnableSequence. Every chain built with LCEL automatically gains invoke, stream, batch, and async variants — no extra code.

ChatPromptTemplate
formats input
|
ChatModel
generates tokens
|
OutputParser
extracts content
RunnablePassthrough
Pass input unchanged — use to inject the raw question into a RAG chain alongside retrieved context.
RunnableLambda
Wrap any Python function as a Runnable. Used for document formatting, preprocessing, or side effects.
RunnableParallel
Run multiple chains simultaneously — results merged into a dict. Both chains share the same input.
PYTHON — LCEL Basics
from langchain_core.prompts import ChatPromptTemplate from langchain_openai import ChatOpenAI from langchain_core.output_parsers import StrOutputParser prompt = ChatPromptTemplate.from_template( "Summarize this in one sentence: {text}" ) model = ChatOpenAI(model="gpt-4o-mini", temperature=0) parser = StrOutputParser() # LCEL chain — same interface for invoke/stream/batch/async chain = prompt | model | parser # Invoke result = chain.invoke({"text": "LangChain is a framework..."}) # Stream for chunk in chain.stream({"text": "LangChain is a framework..."}): print(chunk, end="", flush=True) # Batch (parallel execution) results = chain.batch([{"text": "Doc 1..."}, {"text": "Doc 2..."}])
PYTHON — RunnableParallel
from langchain_core.runnables import RunnableParallel, RunnablePassthrough # Parallel execution — run multiple chains simultaneously parallel_chain = RunnableParallel({ "summary": prompt | model | parser, "original": RunnablePassthrough() }) output = parallel_chain.invoke({"text": "Long document..."}) # output = {"summary": "...", "original": {"text": "Long document..."}}
⚠️
Fallbacks: Use chain.with_fallbacks([backup_chain]) to automatically retry on exceptions. Pairs perfectly with multi-provider strategies — primary Claude, fallback GPT-4o.
03 — RAG

Building a RAG Chain with LCEL

A retriever is itself a Runnable — it accepts a string query and returns a list of Documents. LCEL makes it trivial to compose retrieval, formatting, and generation into a single streamable pipeline.

PYTHON — Full RAG Pipeline
from langchain_community.vectorstores import Chroma from langchain_openai import OpenAIEmbeddings from langchain_core.runnables import RunnableLambda vectorstore = Chroma(embedding_function=OpenAIEmbeddings()) retriever = vectorstore.as_retriever(search_kwargs={"k": 4}) def format_docs(docs): return "\n\n".join(doc.page_content for doc in docs) rag_prompt = ChatPromptTemplate.from_template(""" Answer using only the context below: {context} Question: {question} """) rag_chain = ( {"context": retriever | RunnableLambda(format_docs), "question": RunnablePassthrough()} | rag_prompt | model | parser ) answer = rag_chain.invoke("What is LCEL?")
💡
Context injection pattern: The dict {"context": ..., "question": ...} creates a RunnableParallel implicitly. The retriever runs with the full input string; RunnablePassthrough() passes the same string through as the question.
04 — Observability

LangSmith — Tracing & Observability

LangSmith is Anthropic/LangChain's observability platform. Every Runnable call is automatically traced: you see the full span tree, token counts, latency per step, inputs/outputs, and error messages — without changing any application code.

What Gets Traced
  • → Every Runnable (prompt, model, retriever, parser)
  • → Token counts: input, output, cached
  • → Wall-clock latency per span
  • → Full input/output at each step
  • → Errors with full traceback
Evaluation Runs
  • → Upload a dataset of (input, expected) pairs
  • → Run the chain over the dataset automatically
  • → Score outputs with LLM-as-judge or custom evaluators
  • → A/B compare two prompt versions side-by-side
  • → Track regression across deployments
PYTHON — LangSmith Setup
import os os.environ["LANGCHAIN_TRACING_V2"] = "true" os.environ["LANGCHAIN_API_KEY"] = "ls__..." os.environ["LANGCHAIN_PROJECT"] = "rag-pipeline-v2" # All subsequent chain calls are automatically traced # View at: https://smith.langchain.com result = rag_chain.invoke("What is PagedAttention?") # Trace visible in LangSmith: retriever (4 docs, 12ms), LLM (523 tokens, 1.2s)
05 — Callbacks

Callbacks & Custom Logging

Callbacks let you hook into lifecycle events of every Runnable without modifying chain logic. They are the low-level alternative to LangSmith — useful for custom logging, streaming UI updates, and per-run cost tracking in production.

on_llm_start
Fired before each LLM call — receives prompts and model kwargs.
on_llm_end
Fired after LLM responds — receives full LLMResult including token usage metadata.
on_chain_start/end
Fired at the boundaries of each chain run — use for timing and request correlation IDs.
on_tool_start/end
Fired around tool/retriever invocations — critical for logging which tools agents called.
PYTHON — Custom Token Logger
from langchain_core.callbacks import BaseCallbackHandler class TokenLogger(BaseCallbackHandler): def on_llm_end(self, response, **kwargs): usage = response.llm_output.get("token_usage", {}) print(f"Tokens: {usage.get('total_tokens', 0)}") chain_with_logging = chain.with_config( callbacks=[TokenLogger()] ) result = chain_with_logging.invoke({"text": "..."})
