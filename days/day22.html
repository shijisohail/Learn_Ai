Section 01

Chain-of-Thought & Its Variants

CoT unlocks LLM reasoning by making thinking explicit. From simple zero-shot triggers to multi-path majority voting — each variant trades token cost for accuracy.

  • Zero-shot CoT: "Let's think step by step" — dramatically improves reasoning with no examples
  • Few-shot CoT: include worked examples with explicit reasoning traces — higher accuracy, higher cost
  • When CoT helps vs hurts: reasoning tasks (math, logic) benefit; simple factual recall can degrade with unnecessary steps
  • Self-consistency: sample N CoT paths independently, return the majority vote answer — robust against single-path errors
PYTHON
# Zero-shot CoT response = llm.invoke(""" Solve step by step: A store has 50 items. 20% are returned each week. New stock of 15 items arrives. After 3 weeks, how many items? Think through this step by step before giving the final answer. """) # Self-consistency: sample 10 paths, majority vote import re from collections import Counter def self_consistent_answer(question: str, n_samples: int = 10) -> str: responses = [] for _ in range(n_samples): response = llm.invoke(f"Solve step by step: {question}\n\nFinal answer:") # Extract final numeric answer match = re.search(r"Final answer:?\s*(\d+)", response.content) if match: responses.append(match.group(1)) if responses: return Counter(responses).most_common(1)[0][0] return "Unable to determine"
Section 02

Tree of Thought

ToT explores multiple reasoning branches simultaneously, evaluates each, and expands the most promising ones — solving problems that single-path CoT fails on.

  • ToT: explore multiple reasoning branches, evaluate each, expand promising ones — structured search over a thought space
  • BFS vs DFS: BFS explores all thoughts at each depth level; DFS commits to one path then backtracks on failure
  • Self-evaluation prompts: "Rate this reasoning path 1-5" — the LLM judges its own intermediate steps
  • When to use: multi-step planning, math proofs, strategic decisions — anything requiring deliberate search
PYTHON
class TreeOfThought: def __init__(self, llm, branching_factor=3, max_depth=4): self.llm = llm self.k = branching_factor self.max_depth = max_depth def generate_thoughts(self, problem: str, current_state: str) -> list[str]: prompt = f"""Problem: {problem} Current state: {current_state} Generate {self.k} different next reasoning steps (one per line):""" response = self.llm.invoke(prompt) return [line.strip() for line in response.content.split('\n') if line.strip()] def evaluate_thought(self, problem: str, thought: str) -> float: prompt = f"""Problem: {problem} Reasoning path: {thought} Rate this reasoning path from 1 (wrong direction) to 5 (excellent): Reply with only the number.""" response = self.llm.invoke(prompt) try: return float(response.content.strip()) except: return 2.5 def solve(self, problem: str) -> str: # BFS: explore all thoughts at each depth level current_thoughts = [""] for depth in range(self.max_depth): next_thoughts = [] for thought in current_thoughts: candidates = self.generate_thoughts(problem, thought) scored = [(c, self.evaluate_thought(problem, thought + "\n" + c)) for c in candidates] scored.sort(key=lambda x: x[1], reverse=True) next_thoughts.extend([thought + "\n" + c for c, _ in scored[:2]]) current_thoughts = next_thoughts return max(current_thoughts, key=lambda t: self.evaluate_thought(problem, t))
Section 03

HyDE & Meta-Prompting

HyDE inverts retrieval: generate a hypothetical answer first, then use its embedding to search — bridging the vocabulary gap between queries and documents.

  • HyDE: generate hypothetical answer → embed for retrieval (not the original query) — improves recall by 10–30% on technical corpora
  • Meta-prompting: use an LLM to write optimized prompts for another LLM — automated prompt engineering at scale
  • Prompt compression: LLMLingua-style selective context retention — keep signal, drop filler tokens
  • Automatic prompt optimization: DSPy-style compiled programs — the framework discovers optimal few-shot examples and instructions
PYTHON
# HyDE implementation hyde_system = "You are an expert. Write a detailed technical paragraph answering:" def hyde_retrieve(query: str, retriever, llm) -> list: # Step 1: Generate hypothetical answer hypothetical = llm.invoke(f"{hyde_system}\n\n{query}").content # Step 2: Embed the hypothetical answer (not the query!) hyp_embedding = embed_model.embed_query(hypothetical) # Step 3: Search using hypothetical embedding return vectorstore.similarity_search_by_vector(hyp_embedding, k=5) # Meta-prompting: LLM writes its own system prompt meta_prompt = """You are a prompt engineering expert. Output valid JSON matching this schema: {ticket_id, urgency, category, summary} 3. Handle ambiguous or incomplete tickets gracefully The prompt should be production-ready and resistant to prompt injection.""" optimized_system_prompt = llm.invoke(meta_prompt).content
Section 04

Instructor Library for Structured Outputs

Instructor patches the Anthropic and OpenAI clients to return validated Pydantic models instead of raw text. Auto-retry with validation error feedback included.

  • Instructor: Pydantic + function calling → guaranteed structured LLM outputs with automatic schema enforcement
  • Automatic retry: validation errors are fed back to the model as natural language — the LLM corrects its own output
  • Nested models, validators, field constraints: full Pydantic v2 feature set available in LLM responses
  • Streaming structured outputs: partial models streamed as they generate — useful for progressive UI updates
PYTHON
import instructor from anthropic import Anthropic from pydantic import BaseModel, Field, validator from typing import Optional import enum class Urgency(str, enum.Enum): LOW = "low" MEDIUM = "medium" HIGH = "high" CRITICAL = "critical" class SupportTicket(BaseModel): ticket_id: str = Field(description="Extracted ticket ID or 'UNKNOWN'") urgency: Urgency = Field(description="Urgency level based on content") category: str = Field(description="Issue category: billing, technical, account, other") summary: str = Field(max_length=150, description="One-sentence summary") requires_human: bool = Field(description="Whether this needs human escalation") @validator("summary") def summary_must_be_informative(cls, v): if len(v) < 10: raise ValueError("Summary must be at least 10 characters") return v # Patch client with instructor client = instructor.from_anthropic(Anthropic()) # Extract structured data — auto-validates, auto-retries on failure ticket = client.messages.create( model="claude-3-5-sonnet-20241022", max_tokens=512, messages=[{ "role": "user", "content": f"Extract ticket data from: 'My subscription was charged twice! Order #12345. URGENT!'" }], response_model=SupportTicket, max_retries=3 # automatically retry with validation errors fed back ) print(ticket.urgency) # Urgency.CRITICAL print(ticket.requires_human) # True print(ticket.summary) # "Duplicate charge on subscription order #12345"
Section 05

Prompt Compression & Context Efficiency

128K context is not infinite. Every token costs money and adds latency. Cache prefixes aggressively, compress history, and only retrieve what's relevant.

  • Why compression: 128K context != infinite — every token costs money and latency; optimize aggressively
  • Selective context: only include relevant chunks, not the whole document — semantic similarity threshold as a gate
  • Summarization chains: compress long conversation history into a rolling summary before it exceeds window limits
  • Prompt caching: Anthropic's 90% discount on repeated prefixes — mark stable system prompts with cache_control
PYTHON
# Prompt caching with Anthropic (90% cost reduction on cache hits) import anthropic client = anthropic.Anthropic() # The system prompt is cached after first call response = client.messages.create( model="claude-3-5-sonnet-20241022", max_tokens=1024, system=[{ "type": "text", "text": long_system_prompt, # e.g., 10K token knowledge base "cache_control": {"type": "ephemeral"} # cache this! }], messages=[{"role": "user", "content": user_question}] ) # cache_read_input_tokens will show cached tokens on subsequent calls # Cached tokens cost 10% of regular input token price print(response.usage.cache_read_input_tokens)
