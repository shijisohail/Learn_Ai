Section 01

Chain-of-Thought & Its Variants

CoT unlocks LLM reasoning by making thinking visible and explicit. From simple zero-shot triggers to multi-path majority voting — each variant trades token cost for measurable accuracy gains. Understanding when to use each variant is as important as knowing how they work.

💡
Why this matters: Production evals at Google and Anthropic show CoT improves GPT-4 accuracy on MATH benchmark by up to 40 percentage points. Self-consistency adds another 5–10pp on top. For reasoning-heavy tasks, this is not optional — it's table stakes.

At its core, Chain-of-Thought works because LLMs are autoregressive — each token conditions on all prior tokens. When you force the model to emit reasoning steps, those steps act as scratchpad context that steers subsequent token probabilities toward correct answers. Without explicit reasoning, the model must "collapse" a multi-step derivation into a single token distribution, introducing compression errors.

Self-consistency is an ensemble technique applied at inference time: run the same prompt N times with temperature > 0, parse the final answer from each response, and take the majority vote. It exploits the fact that while individual reasoning paths may err, correct answers cluster more consistently than wrong ones.

  • Zero-shot CoT: "Let's think step by step" — dramatically improves reasoning with zero examples added
  • Few-shot CoT: Include worked examples with explicit reasoning traces — higher accuracy, higher prompt cost
  • When CoT helps vs hurts: reasoning tasks (math, logic, planning) benefit; simple factual recall can degrade with unnecessary steps
  • Self-consistency: Sample N CoT paths independently, return the majority-vote answer — robust against single-path errors
  • Cost vs accuracy tradeoff: Self-consistency with N=10 costs 10x tokens but yields near-oracle accuracy on GSM8K
cot_sampling.py
import re from collections import Counter from anthropic import Anthropic client = Anthropic() def zero_shot_cot(question: str) -> str: """Zero-shot CoT: append the magic trigger phrase.""" msg = client.messages.create( model="claude-3-5-sonnet-20241022", max_tokens=1024, messages=[{"role": "user", "content": question + "\n\nThink step by step before answering."}] ) return msg.content[0].text def self_consistent_answer(question: str, n: int = 5) -> str: """Sample n independent CoT paths, majority-vote the final answer.""" answers = [] for _ in range(n): resp = client.messages.create( model="claude-3-5-sonnet-20241022", max_tokens=800, # temperature > 0 gives diverse paths messages=[{"role": "user", "content": question + "\n\nThink step by step. End with 'Answer: <value>'."}] ) text = resp.content[0].text # extract final answer from response m = re.search(r"Answer:\s*(.+)", text, re.IGNORECASE) if m: answers.append(m.group(1).strip()) if not answers: return "No consistent answer found" # majority vote winner, count = Counter(answers).most_common(1)[0] return f"{winner} (agreed {count}/{n})"
⚠️
Common pitfall: Applying CoT to simple factual lookups like "What is the capital of France?" causes answer degradation. The model gets confused by its own reasoning steps when the answer is direct retrieval, not derivation. Reserve CoT for tasks with non-trivial intermediate steps.
Section 02

Tree of Thought — Branching Exploration

ToT generalizes CoT from a single linear chain to a tree where the model can explore multiple candidate reasoning branches, evaluate them, and backtrack. It turns LLM inference into a heuristic search problem.

Tree of Thought was introduced by Yao et al. (2023) and shows remarkable gains on planning tasks where CoT fails — such as the Game of 24, creative writing with constraints, and multi-step strategic reasoning. The key insight: some problems require looking multiple steps ahead and backtracking when a path is suboptimal. Linear CoT cannot backtrack.

In practice, ToT is implemented as a loop: generate K candidate next-steps (breadth), score each with the model itself (evaluation), keep the top B, and recurse (depth). This is BFS or DFS over the thought-space. Cost is O(B × D) LLM calls for depth D and breadth B, making it expensive but highly accurate on hard tasks.

Section 03

HyDE — Hypothetical Document Embeddings

HyDE improves RAG retrieval by generating a hypothetical answer to the query, embedding that answer (not the query), and using it to find relevant documents. This closes the semantic gap between short queries and rich documents.

Standard dense retrieval computes the embedding of the user query and finds nearest-neighbor documents. This works well when queries and documents are semantically similar in embedding space. But queries are typically short, informal, and lack the technical vocabulary that documents use. HyDE bridges this gap by hallucinating a high-quality hypothetical answer that shares vocabulary and structure with real documents.

The elegance of HyDE is that even if the hypothetical answer is factually wrong, it succeeds in placing the embedding vector in the right neighborhood of the vector space. Retrieval precision and NDCG scores improve across nearly all benchmarks tested in the original Gao et al. (2022) paper.

hyde_retrieval.py
from anthropic import Anthropic from qdrant_client import QdrantClient from sentence_transformers import SentenceTransformer client = Anthropic() qdrant = QdrantClient("http://localhost:6333") encoder = SentenceTransformer("all-MiniLM-L6-v2") def hyde_retrieve(query: str, collection: str, k: int = 5) -> list: """HyDE: generate hypothetical answer → embed it → retrieve.""" # Step 1: Generate hypothetical answer (may be wrong — that's OK) hyp_resp = client.messages.create( model="claude-3-5-haiku-20241022", max_tokens=256, messages=[{ "role": "user", "content": ( f"Write a concise technical answer to: {query}\n" "Respond as if writing a documentation paragraph. Be specific." ) }] ) hypothetical_doc = hyp_resp.content[0].text # Step 2: Embed the hypothetical answer (NOT the raw query) hyp_vector = encoder.encode(hypothetical_doc).tolist() # Step 3: Retrieve against real document store hits = qdrant.search( collection_name=collection, query_vector=hyp_vector, limit=k, with_payload=True ) return [h.payload for h in hits]

Section 04

Instructor + Pydantic for Guaranteed Outputs

Instructor wraps any LLM client to return validated Pydantic models instead of raw strings. It handles retry logic, validation errors, and schema injection automatically — eliminating JSON parsing fragility from your pipelines.

The fundamental problem with LLM outputs in production is that they are free-form text. Even with careful prompting, models occasionally produce invalid JSON, missing fields, or wrong types. Instructor solves this by coupling output generation with Pydantic validation in a retry loop — if the output fails validation, the error is fed back to the model as correction context.

This approach is production-proven at scale: Pydantic's type system provides free documentation, serialization, and error messages that the model can understand and correct. Combined with prompt caching for system prompts, you get both reliability and cost efficiency.

structured_extraction.py
import instructor from pydantic import BaseModel, Field, field_validator from anthropic import Anthropic from typing import Literal # Wrap the Anthropic client with Instructor client = instructor.from_anthropic(Anthropic()) class ExtractedEntity(BaseModel): name: str = Field(description="Entity name as it appears in text") entity_type: Literal["PERSON", "ORG", "LOCATION", "PRODUCT"] confidence: float = Field(ge=0.0, le=1.0) context: str = Field(description="Surrounding context from text") @field_validator('name') @classmethod def name_not_empty(cls, v: str) -> str: if not v.strip(): raise ValueError("Entity name cannot be empty") return v.strip() class ExtractionResult(BaseModel): entities: list[ExtractedEntity] summary: str def extract_entities(text: str) -> ExtractionResult: # Instructor automatically retries on Pydantic validation failures return client.messages.create( model="claude-3-5-sonnet-20241022", max_tokens=1024, response_model=ExtractionResult, max_retries=3, messages=[{ "role": "user", "content": f"Extract all entities from:\n\n{text}" }] ) # Returns a validated ExtractionResult object — no JSON parsing needed result = extract_entities("Apple CEO Tim Cook visited Google HQ in Mountain View.") print(result.entities[0].name) # "Apple" print(result.entities[0].entity_type) # "ORG"

Section 05

Prompt Caching — 90% Cost Reduction

Anthropic's prompt caching allows you to cache large system prompts, documents, and tool definitions. Cached tokens cost 10% of the original price and return 2x faster — critical for document-heavy agents.

💰
90% Cost Saving
Cached input tokens are billed at 10% of standard price. For agents that repeatedly process the same large system prompt or document, savings compound rapidly across conversation turns.
⚡
2× Faster TTFT
Time-to-first-token drops dramatically for cache hits since the KV cache is already warm. Users perceive much lower latency for multi-turn conversations after the first turn.
🎯
Minimum 1024 Tokens
Cache breakpoints require at least 1024 tokens of prefix to activate (2048 for small models). Design your system prompts to front-load stable content before variable content.
prompt_caching.py
from anthropic import Anthropic client = Anthropic() def cached_agent_call(user_message: str, large_document: str) -> str: """Cache the large document once; only charge full price once per 5 min.""" response = client.messages.create( model="claude-3-5-sonnet-20241022", max_tokens=1024, system=[ { "type": "text", "text": "You are an expert document analyst. Be precise and cite sources.", # This large doc is cached — costs 90% less after first call "cache_control": {"type": "ephemeral"} }, { "type": "text", "text": large_document # e.g. 50k token legal contract } ], messages=[{"role": "user", "content": user_message}] ) usage = response.usage print(ff"Cache read: {usage.cache_read_input_tokens} tokens") print(ff"Cache write: {usage.cache_creation_input_tokens} tokens") return response.content[0].text
