Section 01

The ReAct Pattern: Reason + Act

ReAct (Reasoning + Acting) is the foundational pattern for tool-using agents. The model alternates between Thought (reasoning about what to do), Action (calling a tool), and Observation (processing the result) until it reaches a final answer.

The key insight of ReAct is that reasoning and acting are interleaved — not separated. Before each tool call, the model generates a "Thought" explaining its reasoning. This serves two purposes: it improves decision quality by forcing explicit reasoning before action, and it makes the agent's behavior auditable and debuggable. You can literally read the agent's "thinking" in the trace.

💭

Thought — Reasoning

The model reasons about the current state, what information it has, what's missing, and what action to take next. This is where planning and strategy emerge from the base LLM capabilities.

⚡

Action — Tool Call

The model decides to call a tool (search, code executor, API call, calculator). Tool calls are structured JSON that your code handles — they're not LLM generations, they're function invocations.

👁️

Observation — Tool Result

The tool's output is injected back into the context as an "observation." The model processes this new information and begins the next Thought→Action cycle, or determines it has enough to answer.

✅

Final Answer

When the model has gathered enough information through the T→A→O loop, it generates a final answer and stops calling tools. Successful termination requires proper prompt design and tool result handling.

react_agent.py
import anthropic
from typing import Any

client = anthropic.Anthropic()

# Define tools with precise schemas
TOOLS = [
    {
        "name": "web_search",
        "description": "Search the web for current information. Returns top 3 results with snippets.",
        "input_schema": {
            "type": "object",
            "properties": {"query": {"type": "string", "description": "search query"}},
            "required": ["query"]
        }
    },
    {
        "name": "python_exec",
        "description": "Execute Python code in a sandbox. Returns stdout + any errors.",
        "input_schema": {
            "type": "object",
            "properties": {"code": {"type": "string"}},
            "required": ["code"]
        }
    },
    {
        "name": "calculator",
        "description": "Evaluate a mathematical expression. Supports: +,-,*,/,**,%,sqrt,log,sin,cos",
        "input_schema": {
            "type": "object",
            "properties": {"expression": {"type": "string"}},
            "required": ["expression"]
        }
    }
]

def execute_tool(name: str, inputs: dict[str, Any]) -> str:
    """Route tool calls to actual implementations."""
    if name == "web_search":
        return search_web(inputs["query"])  # your search implementation
    elif name == "python_exec":
        return safe_exec(inputs["code"])    # sandboxed Python executor
    elif name == "calculator":
        import math
        return str(eval(inputs["expression"], {"__builtins__": {}}, vars(math)))
    raise ValueError(f"Unknown tool: {name}")

def run_react_agent(user_query: str, max_steps: int = 10) -> str:
    """Run the ReAct loop until completion or max_steps."""
    messages = [{"role": "user", "content": user_query}]

    for step in range(max_steps):
        response = client.messages.create(
            model="claude-sonnet-4-5",
            max_tokens=4096,
            tools=TOOLS,
            messages=messages
        )

        # Append assistant's response to history
        messages.append({"role": "assistant", "content": response.content})

        if response.stop_reason == "end_turn":
            # No tool calls — final answer
            return next(b.text for b in response.content if b.type == "text")

        # Execute all tool calls in this response
        tool_results = []
        for block in response.content:
            if block.type == "tool_use":
                result = execute_tool(block.name, block.input)
                tool_results.append({
                    "type": "tool_result",
                    "tool_use_id": block.id,
                    "content": result
                })

        # Append tool results as user turn for next iteration
        messages.append({"role": "user", "content": tool_results})

    return "Max steps reached without final answer"
⚠️

Always Set max_steps for ReAct Agents

Without a step limit, a poorly-designed agent or a tricky query can cause an infinite loop of tool calls — burning tokens and time. Always set max_steps (usually 5-15 for most tasks, 20-30 for complex research). Also implement cost tracking: log token usage at each step so you can alert if a single agent run exceeds a cost threshold.

Section 03

Multi-Agent Patterns

Single agents have limits — context windows, specialization, parallelism. Multi-agent systems distribute work across specialized agents coordinated by an orchestrator.

The Anthropic multi-agent framework recognizes a key pattern: an orchestrator agent breaks complex tasks into subtasks and delegates to subagent specialists. The orchestrator doesn't need to know how each specialist works — it just calls them as tools via the standard tool_use interface. This composability is what makes multi-agent systems scale.

PatternStructureBest ForComplexity
Single AgentOne LLM + toolsFocused tasks, <15 tool callsLow
Orchestrator-Worker1 orchestrator + N specialistsComplex tasks needing specializationMedium
ParallelN agents in parallel, then mergeIndependent subtasks, researchMedium
PipelineA→B→C sequentialMulti-stage transformationMedium
Debate/CritiqueGenerator + Critic agentsQuality assurance, fact-checkingHigh
orchestrator_agent.py
import anthropic
import asyncio

client = anthropic.Anthropic()

# Specialist agent functions — each has its own system prompt
def research_agent(topic: str) -> str:
    resp = client.messages.create(
        model="claude-sonnet-4-5", max_tokens=1024,
        system="You are a research specialist. Provide factual, sourced information.",
        messages=[{"role": "user", "content": f"Research: {topic}"}]
    )
    return resp.content[0].text

def code_agent(spec: str) -> str:
    resp = client.messages.create(
        model="claude-sonnet-4-5", max_tokens=2048,
        system="You are a Python expert. Write production-ready, tested code only.",
        messages=[{"role": "user", "content": spec}]
    )
    return resp.content[0].text

def critic_agent(content: str) -> str:
    resp = client.messages.create(
        model="claude-sonnet-4-5", max_tokens=512,
        system="You are a critical reviewer. Find flaws, gaps, and improvements. Be concise.",
        messages=[{"role": "user", "content": f"Review this critically:\n{content}"}]
    )
    return resp.content[0].text

# Orchestrator — calls specialists as tools
ORCHESTRATOR_TOOLS = [
    {"name": "delegate_research", "description": "Delegate research to the research specialist",
     "input_schema": {"type": "object", "properties": {"topic": {"type": "string"}}, "required": ["topic"]}},
    {"name": "delegate_coding", "description": "Delegate code writing to the code specialist",
     "input_schema": {"type": "object", "properties": {"spec": {"type": "string"}}, "required": ["spec"]}},
    {"name": "delegate_review", "description": "Have the critic agent review content",
     "input_schema": {"type": "object", "properties": {"content": {"type": "string"}}, "required": ["content"]}}
]

def orchestrate(task: str) -> str:
    messages = [{"role": "user", "content": task}]
    while True:
        resp = client.messages.create(
            model="claude-sonnet-4-5", max_tokens=2048,
            system="Break complex tasks into subtasks. Delegate to specialists. Synthesize results.",
            tools=ORCHESTRATOR_TOOLS, messages=messages
        )
        messages.append({"role": "assistant", "content": resp.content})
        if resp.stop_reason == "end_turn":
            return next(b.text for b in resp.content if b.type == "text")
        results = []
        for block in resp.content:
            if block.type == "tool_use":
                r = {"research": research_agent, "coding": code_agent, "review": critic_agent}
                fn_key = block.name.replace("delegate_", "")
                result = list(block.input.values())[0]
                output = r[fn_key](result)
                results.append({"type": "tool_result", "tool_use_id": block.id, "content": output})
        messages.append({"role": "user", "content": results})
