02 — OpenAI

OpenAI API — GPT-4o & Structured Outputs

OpenAI's key differentiator is ecosystem depth: the largest library of integrations, the most battle-tested function calling, and Structured Outputs — guaranteed JSON Schema adherence via constrained decoding, not prompting tricks.

Structured Outputs

Guarantees the response matches your Pydantic schema exactly — no hallucinated keys, no type mismatches. Uses constrained decoding at the token level, not post-processing.

Use over JSON mode when: schema correctness is critical (extraction pipelines, structured data for DB inserts).
Batch API

Submit up to 50,000 requests asynchronously, results within 24h, at 50% cost discount. Ideal for offline workloads: nightly evals, bulk document classification, dataset enrichment.

When not to use: real-time user-facing features where latency matters.
PYTHON — Structured Outputs
from openai import OpenAI from pydantic import BaseModel client = OpenAI() class ExtractedData(BaseModel): title: str summary: str key_entities: list[str] sentiment: str # "positive" | "neutral" | "negative" # Structured Outputs — guaranteed schema compliance completion = client.beta.chat.completions.parse( model="gpt-4o-2024-11-20", messages=[{"role": "user", "content": f"Extract: {document}"}], response_format=ExtractedData, ) data: ExtractedData = completion.choices[0].message.parsed print(data.title, data.key_entities)
03 — Anthropic

Anthropic Claude API — Extended Context & Tool Use

Claude's 200K context window is the largest among frontier models (as of 2025). Combined with prompt caching for repeated system prompts and a clean tool use API, it's the top choice for document intelligence and agentic applications.

Prompt Caching

Mark system prompt blocks with cache_control: {"type": "ephemeral"}. Anthropic caches the prefix KV — subsequent calls re-use the cached computation.

Cache hit: ~90% cost reduction on cached tokens
Extended Thinking

Enable with thinking: {"type": "enabled", "budget_tokens": 8000}. Claude reasons step-by-step in a hidden scratchpad before producing the final answer.

Best for: multi-step math, complex code architecture decisions, hard reasoning tasks.
PYTHON — Claude Tool Use
import anthropic client = anthropic.Anthropic() # Tool use with Claude tools = [{ "name": "search_database", "description": "Search the product database", "input_schema": { "type": "object", "properties": { "query": {"type": "string"}, "limit": {"type": "integer", "default": 10} }, "required": ["query"] } }] response = client.messages.create( model="claude-3-5-sonnet-20241022", max_tokens=1024, tools=tools, messages=[{"role": "user", "content": "Find products under $50"}] ) if response.stop_reason == "tool_use": tool_call = next(b for b in response.content if b.type == "tool_use") # tool_call.name = "search_database" # tool_call.input = {"query": "products under $50", "limit": 10}
04 — Open-Weight

Google Gemini & Open-Weight Models

Gemini's 1M context window enables use cases impossible with other providers: full codebase analysis, hour-long video understanding, entire book summarisation in a single call. Open-weight models like Llama 3 and Mistral unlock full data sovereignty and zero inference cost at scale.

Gemini 1.5 Pro — 1M Context
  • → 1M tokens ≈ ~700K words, or ~30,000 lines of code
  • → Native video, audio, image, and code understanding
  • → Google AI Studio (prototyping) vs Vertex AI (enterprise, VPC)
  • → Caveat: cost at 1M context is significant — price per call
When to Self-Host
  • → Data sovereignty — PII, PHI, or classified data
  • → Cost at scale — millions of calls/day makes inference cheap
  • → Custom fine-tuned models (not available via APIs)
  • → Stack: vLLM (high-throughput) or Ollama (local dev)
PYTHON — Mistral + Llama via Ollama
# Mistral — same OpenAI-compatible interface from mistralai import Mistral client = Mistral(api_key="...") response = client.chat.complete( model="mistral-large-latest", messages=[{"role": "user", "content": "Explain LoRA"}] ) # Llama 3 via Ollama (local, OpenAI-compatible) from openai import OpenAI local_client = OpenAI(base_url="http://localhost:11434/v1", api_key="ollama") response = local_client.chat.completions.create( model="llama3.2:3b", messages=[{"role": "user", "content": "Explain LoRA"}] )
05 — Strategy

Multi-Provider Strategy & Fallback Patterns

Production systems should never depend on a single provider. A robust fallback chain plus a cost routing layer gives you resilience and optimal spend — no rewrite required when providers have outages.

💡
Cost routing pattern: Classify query complexity first (fast, cheap model). Route simple intent/classification to GPT-4o-mini or Haiku. Only escalate to frontier (GPT-4o, Claude 3.5 Sonnet) for complex reasoning. Reduces cost by 60–80% in typical RAG apps.
PYTHON — LangChain Fallback Chain
from langchain_anthropic import ChatAnthropic from langchain_openai import ChatOpenAI # Fallback chain: Claude → GPT-4o → GPT-4o-mini primary = ChatAnthropic(model="claude-3-5-sonnet-20241022") secondary = ChatOpenAI(model="gpt-4o") fallback = ChatOpenAI(model="gpt-4o-mini") # LangChain fallback chain — tries each in order on exception robust_llm = primary.with_fallbacks([secondary, fallback]) result = robust_llm.invoke("Explain RLHF")
⚠️
Circuit breaker pattern: Track failure rate per provider in a sliding window. If failure rate exceeds threshold (e.g. 50% in 60s), open the circuit and skip that provider entirely for a cool-down period. Re-close on successful probe. Prevents cascading slowdowns from timeout accumulation.
