Section 02

The Fine-Tuning Phase

QLoRA-fine-tune Llama-3.2-3B on a domain-specific Q&A dataset. Every training run is logged to MLflow — hyperparams, loss curves, adapter artifacts, and final eval metrics.

📋
Dataset Format (Alpaca): Each example needs "instruction", "input", and "output" keys. For customer support: instruction = "Answer this support question.", input = customer query, output = ideal agent response. Minimum 500 examples for meaningful fine-tuning.
PYTHON — Complete Training Script with MLflow
import mlflow import torch from transformers import AutoModelForCausalLM, AutoTokenizer, BitsAndBytesConfig from peft import LoraConfig, get_peft_model, prepare_model_for_kbit_training, PeftModel from trl import SFTTrainer, SFTConfig from datasets import load_dataset mlflow.set_experiment("week3-capstone-qlora") with mlflow.start_run(run_name="llama3-3b-domain-ft"): # Log config mlflow.log_params({"model": "Llama-3.2-3B", "rank": 16, "dataset": "domain-qa"}) # QLoRA setup bnb_config = BitsAndBytesConfig( load_in_4bit=True, bnb_4bit_quant_type="nf4", bnb_4bit_compute_dtype=torch.bfloat16 ) model = AutoModelForCausalLM.from_pretrained( "meta-llama/Llama-3.2-3B", quantization_config=bnb_config, device_map="auto" ) model = prepare_model_for_kbit_training(model) model = get_peft_model(model, LoraConfig(r=16, lora_alpha=32, task_type="CAUSAL_LM")) # Train trainer = SFTTrainer(model=model, train_dataset=dataset, tokenizer=tokenizer, args=SFTConfig(num_train_epochs=3, output_dir="./ckpts")) trainer.train() # Log final metrics mlflow.log_metrics({"eval_loss": trainer.evaluate()["eval_loss"]}) trainer.save_model("./lora-adapter")
Section 03

FastAPI Serving

Load the merged model into a FastAPI app. Async endpoints, health check with model metadata, and authentication middleware. This is the API that goes behind your load balancer.

⚡
Async Endpoints
Use async def and asyncio.to_thread() to avoid blocking the event loop during GPU inference. FastAPI serves multiple connections while one inference runs.
📡
Streaming SSE
Server-Sent Events for streaming token-by-token output. Use StreamingResponse with a generator. Reduces perceived latency by 60-70% on long outputs.
🔐
Auth Middleware
API key validation as a FastAPI dependency. Inject it into protected endpoints with Depends(verify_api_key). Rate limiting via slowapi or Redis.
❤️
Health Endpoint
GET /health returns model name, version, GPU memory used, and uptime. Kubernetes liveness and readiness probes hit this. Essential for zero-downtime deploys.
PYTHON — FastAPI Serving
from fastapi import FastAPI, HTTPException from fastapi.responses import StreamingResponse from pydantic import BaseModel import asyncio app = FastAPI(title="Domain LLM API") class GenerateRequest(BaseModel): prompt: str max_tokens: int = 200 temperature: float = 0.7 @app.get("/health") async def health(): return {"status": "ok", "model": "llama3-3b-domain-ft", "version": "1.0.0"} @app.post("/generate") async def generate(request: GenerateRequest): inputs = tokenizer(request.prompt, return_tensors="pt").to("cuda") with torch.no_grad(): outputs = model.generate( **inputs, max_new_tokens=request.max_tokens, temperature=request.temperature, do_sample=True ) new_tokens = outputs[0][inputs["input_ids"].shape[1]:] response_text = tokenizer.decode(new_tokens, skip_special_tokens=True) return {"response": response_text}
Section 04

RAGAS Evaluation

Build a golden eval dataset for your domain. Run RAGAS against the fine-tuned model's RAG pipeline and compare scores against the base model. Log everything to MLflow.

🏆
What "Good" Looks Like: A well-fine-tuned domain model should show: faithfulness > 0.85 (base: ~0.78), answer relevancy > 0.88 (base: ~0.80), and context utilization > 0.80 (base: ~0.72). If fine-tuning doesn't improve these, check your dataset quality first.

Answer Correctness 0.69 0.82 +18.8%
