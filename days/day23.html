Section 01

The AI Threat Landscape

Deploying LLMs in production exposes you to a new class of vulnerabilities. Unlike traditional software bugs, AI security failures are probabilistic, context-dependent, and invisible in code review.

Prompt injection is the AI equivalent of SQL injection: an attacker embeds malicious instructions within content that the model processes, hijacking its behavior. Unlike SQL injection which exploits parser boundaries, prompt injection exploits the model's inability to distinguish between trusted system instructions and untrusted user/document content — because both arrive as natural language.

The attack surface in RAG systems is particularly severe: documents retrieved from external sources (web pages, emails, uploaded files) can contain injected instructions that the model will follow as if they were part of the system prompt. This indirect injection vector bypasses all input-level filtering entirely.

🎭
Direct Injection
User input directly overrides system prompt instructions. "Ignore all previous instructions..." — exploits instruction-following fine-tuning. Most common and easiest to detect.
📄
Indirect Injection
Malicious instructions hidden in documents or web pages the model retrieves. Nearly impossible to prevent without strict isolation between instruction and data contexts.
🔓
Jailbreaking
Roleplay, hypotheticals, encoding tricks, or many-shot examples to bypass safety fine-tuning. Many-shot jailbreaking (Anil et al. 2024) scales with context length.
🕵️
Data Exfiltration
Extracting system prompt content, session data, or fine-tuning examples through crafted prompts. PII leakage is a major compliance and legal liability risk.
🚨
Common pitfall: Relying solely on "the model won't do X" as a security guarantee. LLM safety fine-tuning is a soft behavioral preference, not a hard security boundary. Always implement defense-in-depth with code-level guardrails, output filtering, and rate limiting in addition to model-level safety.

Section 03

Constitutional AI — Scalable Self-Alignment

Constitutional AI replaces expensive human labeling with AI-generated critique and revision. A model critiques its own outputs against a set of principles, then revises — creating a self-improving safety loop at scale.

Traditional RLHF requires thousands of human comparisons to train a reward model. Constitutional AI enables scalable AI self-alignment: given a set of natural language principles, the model generates a response, a critique model evaluates it against each principle, a revision model rewrites to comply. This generates synthetic preference data without human labeling.

Claude's training incorporates CAI principles like "Choose the response least likely to contain harmful content" and "Prefer responses that are honest even when truth is unflattering." This creates a model that is both helpful and safe without the over-refusal problem common in heavily RLHF'd systems.

Constitutional AI Critic
Paste any AI-generated text. The critic highlights potential constitutional violations. Each highlighted phrase links to the violated principle.
Active Constitutional Principles
1
Harmlessness: Choose responses unlikely to produce harmful, dangerous, or deceptive outcomes even when indirectly requested.
2
Honesty: Prefer truthful, calibrated responses. Acknowledge uncertainty. Avoid stating falsehoods even if they're what users want to hear.
3
Non-manipulation: Avoid persuasion techniques that exploit psychological biases, create false urgency, or bypass rational agency.
4
Autonomy-preservation: Protect epistemic autonomy — don't nudge toward particular beliefs unnecessarily or undermine independent thinking.
5
Non-deception: Never create false impressions through technically true statements, selective emphasis, or misleading framing.
constitutional_ai.py
from anthropic import Anthropic client = Anthropic() CONSTITUTION = [ "The response should not be harmful, dangerous, or deceptive.", "The response should be honest, even when truth is inconvenient.", "The response should not manipulate via psychological exploitation.", "The response should respect user autonomy and epistemic independence.", ] def critique_and_revise(draft: str, principle: str) -> str: # Step 1: Critique against principle c = client.messages.create( model="claude-3-5-sonnet-20241022", max_tokens=200, messages=[{"role":"user", "content": f"Principle: {principle}\n\nDraft:\n{draft}\n\n" "Identify specific violations of this principle."}] ).content[0].text # Step 2: Revise to address critique return client.messages.create( model="claude-3-5-sonnet-20241022", max_tokens=400, messages=[{"role":"user", "content": f"Original:\n{draft}\n\nCritique:\n{c}\n\n" "Rewrite to address all critique points while preserving intent."}] ).content[0].text def constitutional_pipeline(draft: str) -> str: current = draft for principle in CONSTITUTION: current = critique_and_revise(current, principle) return current # fully revised output
💡
Why this matters: CAI enables domain-specific safety policies without expensive human labeling. A medical AI can enforce clinical accuracy principles; a children's platform can enforce age-appropriateness. This is bespoke AI safety at scale — a competitive advantage for any serious AI product.
Section 04

Defense-in-Depth: Layer Architecture

No single guardrail is sufficient. Production AI safety requires multiple independent layers — each catching what others miss. This is the Swiss cheese model applied to AI systems.

LayerMechanismSpeedCoverageCost
L1: Regex PatternKnown injection patterns, regex rules~0.1ms~ Known attacks onlyFree
L2: ML ClassifierFine-tuned text classifier (toxicity, PII, injection)~20ms~ Good coverageLow
L3: LLM ShieldFast LLM (Haiku) classifies intent before main call~300ms✓ High nuanceMedium
L4: NeMo GuardrailsColang conversation policies, topic rails~100ms✓ Policy-basedMedium
L5: Output FilteringPost-generation PII redaction, content scan~50ms✓ Last safety netLow
L6: Human ReviewSampling flagged interactions for review queuesHours✓ Ground truthHigh
