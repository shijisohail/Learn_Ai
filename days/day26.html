01 — Strategy

When to Use Managed AI Services

Self-hosted vs managed is not a technical decision — it's an economics and risk decision. The answer depends on your volume, compliance burden, and team size.

💡
Rule of thumb: Below ~100M tokens/month, managed services almost always win on TCO. Above that threshold, compare carefully — GPU amortization can flip the math.
🖥️
Self-Hosted (vLLM / Ollama)
Maximum control over model weights and data. Cost-efficient at massive scale (>100M tokens/month). Required for custom fine-tuned models. Full data sovereignty — nothing leaves your infra.
☁️
Managed (Bedrock / Vertex / Azure)
Zero infrastructure management. Pay-per-token with no upfront GPU cost. Enterprise compliance built-in (HIPAA, SOC2, FedRAMP). Scales instantly to handle variable workloads.
⚖️
Decision Matrix
Cost vs compliance vs control vs time-to-market. Small team + compliance requirements → managed wins. Large team + predictable high volume + custom models → self-hosted wins.
🔀
Hybrid Approach
Use Bedrock/Vertex for proprietary frontier models (Claude, GPT-4o). Use vLLM self-hosted for high-volume open models (Llama, Mistral). Route by cost, latency, and sensitivity.

02 — AWS Bedrock

AWS Bedrock

Amazon's managed AI service gives you access to frontier models through your existing AWS infrastructure — IAM, VPCs, CloudTrail, and all the compliance you already have.

  • Supported models: Claude (Anthropic), Nova/Titan (Amazon), Llama (Meta), Mistral, Stable Diffusion — 30+ models via one API
  • Cross-region inference: Route requests to regions with available capacity — prevents throttling during peak load
  • Bedrock Agents: Managed agent orchestration with memory, knowledge bases, and action groups — Bedrock's alternative to LangGraph
  • Compliance: HIPAA, SOC 2 Type II, FedRAMP High, ISO 27001 — data never leaves your AWS account, encrypted at rest and in transit
🔐
Security note: Bedrock uses AWS IAM for auth — no API keys in code. Attach an IAM role to your EC2/Lambda/ECS and use boto3 with instance credentials. Your requests are not used for model training.
PYTHON — AWS Bedrock (Claude)
import boto3 import json bedrock = boto3.client("bedrock-runtime", region_name="us-east-1") # Invoke Claude via Bedrock response = bedrock.invoke_model( modelId="anthropic.claude-3-5-sonnet-20241022-v2:0", contentType="application/json", accept="application/json", body=json.dumps({ "anthropic_version": "bedrock-2023-05-31", "max_tokens": 1024, "messages": [ {"role": "user", "content": "Explain LoRA fine-tuning"} ] }) ) result = json.loads(response["body"].read()) print(result["content"][0]["text"]) # Streaming response response_stream = bedrock.invoke_model_with_response_stream( modelId="anthropic.claude-3-5-sonnet-20241022-v2:0", body=json.dumps({ "anthropic_version": "bedrock-2023-05-31", "max_tokens": 512, "messages": [{"role": "user", "content": "Explain attention"}] }) ) for event in response_stream["body"]: chunk = json.loads(event["chunk"]["bytes"]) if chunk.get("type") == "content_block_delta": print(chunk["delta"]["text"], end="", flush=True)
03 — Google Vertex AI

Google Vertex AI

Vertex gives you Gemini on a private enterprise endpoint with GCP's IAM, VPC, and data residency controls. Plus Model Garden for fine-tuning open models on managed infrastructure.

  • Gemini on Vertex: Private endpoint with VPC Service Controls, IAM auth (no API keys exposed), and audit logging in Cloud Audit Logs
  • Model Garden: Fine-tune Gemma, Llama 3, and Mistral on Vertex-managed TPUs/GPUs — no cluster to manage
  • Vertex AI Agent Builder: Managed RAG, enterprise search, and conversational agents with grounding in Google Search or your own data
  • Vertex AI Pipelines: Kubeflow-compatible managed ML pipelines for training, eval, and deployment — integrates with MLflow
PYTHON — Vertex AI (Gemini)
import vertexai from vertexai.generative_models import GenerativeModel, Part, SafetySetting vertexai.init(project="my-gcp-project", location="us-central1") model = GenerativeModel( "gemini-1.5-pro-002", system_instruction="You are an expert AI engineer.", safety_settings=[ SafetySetting( category=SafetySetting.HarmCategory.HARM_CATEGORY_HATE_SPEECH, threshold=SafetySetting.HarmBlockThreshold.BLOCK_LOW_AND_ABOVE ) ] ) # Multi-modal: text + image/video response = model.generate_content([ Part.from_uri("gs://bucket/video.mp4", mime_type="video/mp4"), "Summarize the key points in this video" ]) # Streaming for chunk in model.generate_content("Explain LoRA", stream=True): print(chunk.text, end="", flush=True) # LangChain integration from langchain_google_vertexai import ChatVertexAI llm = ChatVertexAI(model="gemini-1.5-pro", location="us-central1")
04 — Azure OpenAI

Azure OpenAI Service

Microsoft hosts OpenAI's models in your Azure tenant. Same OpenAI API surface, but with Azure AD auth, private endpoints, CMEK, and enterprise SLAs — making it the choice for regulated industries already on Azure.

  • Azure-hosted OpenAI models: GPT-4o, o1, o3-mini, DALL-E 3, Whisper, text-embedding-3-large — same models, Azure compliance wrapper
  • Enterprise features: Private endpoints, Customer-Managed Encryption Keys (CMEK), Azure AD/Entra auth, configurable content filters
  • Azure AI Content Safety: Configurable content filters with custom categories — layer on top of model output for regulated content
  • Azure AI Foundry (formerly AI Studio): Central hub for all Azure AI services — manage deployments, fine-tuning, and evals from one place
✅
Production best practice: Never use API keys in Azure production. Use DefaultAzureCredential() with a Managed Identity — zero secrets to rotate, full audit trail in Azure Monitor.
PYTHON — Azure OpenAI (with Managed Identity)
from openai import AzureOpenAI # Dev: API key (never in production) client = AzureOpenAI( azure_endpoint="https://my-company.openai.azure.com/", api_key="...", api_version="2024-10-01-preview" ) # Same OpenAI API, different endpoint response = client.chat.completions.create( model="gpt-4o", # deployment name in Azure messages=[{"role": "user", "content": "Explain LoRA"}], max_tokens=500 ) # Production: Managed Identity (no API keys) from azure.identity import DefaultAzureCredential, get_bearer_token_provider from openai import AzureOpenAI credential = DefaultAzureCredential() token_provider = get_bearer_token_provider( credential, "https://cognitiveservices.azure.com/.default" ) client = AzureOpenAI( azure_endpoint="https://my-company.openai.azure.com/", azure_ad_token_provider=token_provider, api_version="2024-10-01-preview" )
