Autograd & Gradient Computation

Understand how PyTorch builds the computation graph and computes gradients — the engine behind fine-tuning.

Completed
Your Task

Implement a manual gradient descent step using autograd:

  • Compute a simple loss (e.g., loss = (w * x - y).pow(2).mean())
  • Call loss.backward() and inspect w.grad

w.grad.zero_() — the trailing underscore means in-place. Calling loss.backward() accumulates gradients by default; you must zero between steps.

import torch import torch.nn as nn N = 1024 t32 = torch.randn(N, N, dtype=torch.float32) # TODO: create t16 (float16) and tbf (bfloat16) t16 = ... tbf = ... print(f"float32 : {t32.nbytes / 1024**2:.1f} MB") print(f"float16 : {t16.nbytes / 1024**2:.1f} MB") print(f"bfloat16 : {tbf.nbytes / 1024**2:.1f} MB") # Matmul in each dtype out32 = t32 @ t32.T # TODO: matmul for t16 and tbf ... # Model size utility def model_size_mb(model: nn.Module) -> float: """Return total parameter memory in MB.""" # TODO: sum p.nbytes for p in model.parameters() ... model = nn.Linear(4096, 4096) # 64MB in float32 print(f"\nLinear(4096,4096) float32: {model_size_mb(model):.1f} MB") print(f"Linear(4096,4096) float16: {model_size_mb(model.half()):.1f} MB")
bfloat16 vs float16: Both are 16-bit (2 MB per 1M params vs 4 MB for fp32). bfloat16 has the same exponent range as float32 — far less likely to overflow during training. That's why Llama, Mistral, etc. default to bf16.
Test Cases
T1

→ True (float32 uses 4 bytes, float16 uses 2)
T2

→ True (both 16-bit, same byte count)
T3

→ True (4096*4096*4 bytes ≈ 64 MB)
T4
(t32 @ t32.T).shape == (t16 @ t16.T).shape
→ True (output shape independent of dtype)
Progressive Hints
t16 = torch.randn(N, N, dtype=torch.float16) or t16 = t32.half(). For bfloat16: t32.to(torch.bfloat16).
return sum(p.nbytes for p in model.parameters()) / 1024**2. Include buffers too if you want full accuracy: model.buffers().
