Your First LCEL Chain

Build a prompt → model → parser chain using the pipe operator and understand how runnables compose.

Completed
Objective
Goal: Use LCEL's | pipe operator to compose a ChatPromptTemplate → ChatOpenAI (or Claude) → StrOutputParser chain. Invoke it synchronously, then stream it. Understand that every component is a Runnable with .invoke(), .stream(), and .batch().
Your Task

Build a composable joke chain:

  • Create a ChatPromptTemplate with a system message and a user variable {topic}
  • Pipe it through an LLM and StrOutputParser
  • Invoke with chain.invoke({"topic": "Python"}) and print the result
  • Stream the same chain with chain.stream(...) and print each chunk as it arrives
  • Use chain.batch([{"topic": "Docker"}, {"topic": "Git"}]) and print both results
Starter Code
ex1_lcel_basics.py Python 3.11+

Define the model — TODO: swap to ChatAnthropic if preferred llm = ChatOpenAI(model="gpt-4o-mini", temperature=0.7) # 3. Build the chain using the pipe operator chain = prompt | llm | StrOutputParser() # TODO: invoke synchronously result = ... print("Invoke:", result) # TODO: stream and print each chunk print("\nStream:") for chunk in ...: print(chunk, end="", flush=True) # TODO: batch two topics results = ... for r in results: print("\nBatch:", r)

Hints
prompt | llm | parser creates a RunnableSequence. Each component's output feeds into the next's input. The pipe operator calls .__or__() under the hood. You can inspect the chain type with type(chain).
With StrOutputParser, each chunk from .stream() is already a str. Without the parser, chunks are AIMessageChunk objects — access .content to get the text.
chain.batch(inputs) runs all inputs concurrently using a thread pool by default. You can control concurrency with chain.batch(inputs, config={"max_concurrency": 2}).

Completed

Exercise 3 of 6

Memory & Conversation History

Add chat memory to a chain so it remembers previous turns in a multi-turn conversation.

Completed
Objective
Goal: Use ChatMessageHistory and RunnableWithMessageHistory to add persistent conversation memory to an LCEL chain. The model should be able to reference what was said in earlier turns without you manually managing the history list.
Your Task

Build a stateful chatbot:

  • Create a ChatPromptTemplate that includes MessagesPlaceholder("history")
  • Wrap your chain with RunnableWithMessageHistory and a session-keyed store
  • Send three messages in the same session: introduce yourself, ask a follow-up, verify memory
  • Send a message in a different session and confirm it starts fresh with no memory of the first session
  • Print the full stored history after 3 turns
Starter Code
ex3_memory.py Python 3.11+
from langchain_core.prompts import ChatPromptTemplate, MessagesPlaceholder from langchain_core.output_parsers import StrOutputParser from langchain_community.chat_message_histories import ChatMessageHistory from langchain_core.runnables.history import RunnableWithMessageHistory from langchain_openai import ChatOpenAI llm = ChatOpenAI(model="gpt-4o-mini") store: dict[str, ChatMessageHistory] = {} def get_history(session_id: str) -> ChatMessageHistory: if session_id not in store: store[session_id] = ChatMessageHistory() return store[session_id] # TODO: build the prompt with MessagesPlaceholder prompt = ChatPromptTemplate.from_messages([ ("system", "You are a helpful assistant with a great memory."), MessagesPlaceholder(variable_name="history"), ("user", "{input}"), ]) chain = prompt | llm | StrOutputParser() # TODO: wrap with RunnableWithMessageHistory chain_with_history = RunnableWithMessageHistory( chain, get_history, input_messages_key="input", history_messages_key="history", ) cfg = {"configurable": {"session_id": "session-1"}} # TODO: send 3 messages in session-1 r1 = chain_with_history.invoke({"input": "My name is Sharjeel and I love Python."}, cfg) r2 = chain_with_history.invoke({"input": "What's my name?"}, cfg) r3 = chain_with_history.invoke({"input": "What language did I say I love?"}, cfg) print(r2) # Should mention "Sharjeel" print(r3) # Should mention "Python" # TODO: start session-2 and verify it's fresh r_new = chain_with_history.invoke( {"input": "What's my name?"}, {"configurable": {"session_id": "session-2"}} ) print("Session 2:", r_new) # Should not know the name
Expected Output
TODO
r2 (session-1, turn 2)
→ Response contains "Sharjeel"
TODO
r_new (session-2, turn 1)
→ Does NOT know the name — fresh context
Hints
It intercepts .invoke(), calls get_history(session_id), injects the messages into the prompt's MessagesPlaceholder, then after getting a response automatically appends both the user message and AI response to the history store.
store["session-1"].messages gives you the full list of HumanMessage and AIMessage objects. Each has a .content attribute.

Objective

Your Task

Completed
Objective
Goal: Enable LangSmith tracing with environment variables, add custom metadata tags, create a nested trace by calling a chain inside another chain, and use @traceable decorator to trace a plain Python function. View all traces in the LangSmith dashboard.
Your Task

LCEL Router Chain

Build a dynamic router that classifies user input and dispatches to the right specialist chain — without any if/else in the outer code.

Completed
Objective
Goal: Use RunnableBranch (or RunnableLambda with dispatch logic) to build a router that detects whether the user wants code help, math help, or general help, then routes to the specialist chain. Implement fallback for unknown topics.
Your Task

Build an intent-routing pipeline:

  • Create a classifier chain that returns "code", "math", or "general" given any user question
  • Create three specialist prompts: code expert, math tutor, general assistant
  • Use RunnableBranch to route based on the classifier output
  • Test with: a Python debugging question, a calculus question, and a recipe question
  • Log which branch was selected for each query
Starter Code
ex6_router.py Python 3.11+

Answer: {question}" ) | llm | parser math_chain = ... # TODO: math tutor prompt general_chain = ... # TODO: general assistant prompt # TODO: build router with RunnableBranch router = RunnableBranch( (lambda x: x["category"] == "code", code_chain), (lambda x: x["category"] == "math", math_chain), general_chain, # fallback ) # Full pipeline: classify then route pipeline = { "question": RunnablePassthrough(), "category": classifier, } | RunnableParallel( # keep question accessible answer=router, category=RunnableLambda(lambda x: x["category"]) ) questions = [ "Why does my Python list comprehension raise a TypeError?", "How do I integrate x^2 from 0 to 3?", "What's a good recipe for banana bread?", ] for q in questions: out = pipeline.invoke(q) print(f"[{out['category']}] {out['answer'][:80]}...")

Hints
RunnableBranch passes the entire input dict to each condition lambda. Your specialist chains also receive this dict — make sure their prompt templates reference the right key (e.g., {question}).
You can also use a RunnableLambda that looks up the category in a dict: {"code": code_chain, "math": math_chain}.get(x["category"], general_chain).invoke(x). This is often cleaner than RunnableBranch.
