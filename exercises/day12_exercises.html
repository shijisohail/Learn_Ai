Hybrid Search: BM25 + Vector with RRF

Combine keyword search and semantic search using Reciprocal Rank Fusion for better retrieval.

Completed
Objective
Goal: Pure vector search fails on exact keyword queries (e.g. error codes, function names). Implement BM25 for keyword retrieval, vector search for semantic retrieval, then fuse their ranked lists using Reciprocal Rank Fusion (RRF) — the simple formula that beats expensive re-rankers on many benchmarks.
Your Task

Build a hybrid search pipeline:

  • Install rank-bm25 and implement a BM25 retriever returning top-k (index, score) pairs
  • Implement a vector retriever using sentence-transformers + cosine similarity
  • Implement rrf_fusion(ranked_lists, k=60) — RRF score = sum of 1/(k + rank) across lists
  • Test on a corpus with 3 queries: one exact-keyword, one semantic, one mixed
  • Compare: which retriever wins on each query type? Does RRF beat both individually?

HyDE — Hypothetical Document Embeddings

Improve retrieval for abstract queries by generating a hypothetical answer first, then embedding it.

Completed
Objective
Goal: HyDE works by asking an LLM to write a hypothetical answer to the query, then using that answer's embedding (instead of the query's embedding) to search the corpus. The hypothesis is often more similar to real documents than the raw question.
Your Task

Implement and compare HyDE vs standard retrieval:

  • Write generate_hypothesis(query: str, llm) -> str — prompt the LLM to write a 1-paragraph hypothetical document answering the query
  • Embed the hypothesis (not the query) and use it to search the corpus
  • Compare top-3 results: standard query embedding vs HyDE embedding
  • Test on 3 abstract queries where HyDE should help (e.g. "best practices for API security")
  • Log the generated hypothesis for each query to understand what the LLM produced

Cross-Encoder Re-Ranking

Use a cross-encoder model to re-score top-k retrieved documents and improve precision.

Completed
Objective
Goal: Bi-encoder (embedding) retrieval is fast but approximate. Cross-encoders see both query and document jointly and score relevance more accurately — but are 100x slower. The two-stage pattern: retrieve 20 with bi-encoder, re-rank with cross-encoder, return top-5.
Your Task

Build a two-stage retrieval pipeline:

  • Retrieve top-20 candidates using bi-encoder (sentence-transformers)
  • Re-rank with CrossEncoder("cross-encoder/ms-marco-MiniLM-L-6-v2")
  • Compare final top-5 before and after re-ranking
  • Measure latency of each stage: bi-encoder retrieval vs cross-encoder re-ranking
  • Test on 3 queries and verify the re-ranker improves or maintains quality

RAGAS Evaluation — Faithfulness & Relevancy

Automatically evaluate your RAG pipeline using RAGAS metrics against a golden Q&A dataset.

Completed
Objective
Goal: Use ragas to evaluate a RAG pipeline on three key metrics: Faithfulness (is the answer supported by the retrieved context?), Answer Relevancy (does the answer address the question?), and Context Recall (did retrieval fetch the right chunks?).
Your Task

Run RAGAS evaluation on a RAG pipeline:

  • Build a simple RAG pipeline: retrieve 3 docs from ChromaDB, generate answer with GPT-4o-mini
  • Create a golden dataset of 5 Q&A pairs with ground-truth answers
  • Format results as a Dataset with columns: question, answer, contexts, ground_truth
  • Run ragas.evaluate(dataset, metrics=[faithfulness, answer_relevancy])
  • Print the scores and identify which Q&A pairs scored lowest on faithfulness
Starter Code
