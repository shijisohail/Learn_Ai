Objective

Create an intent classifier that takes a user message and returns a structured classification with intent, extracted entities, confidence, and clarification flag.

  • Define ClassifiedIntent Pydantic model with: intent, entities (list[str]), confidence (float 0-1), needs_clarification (bool)
  • Implement classify_intent(message: str) -> ClassifiedIntent using instructor or JSON mode
  • Test with 5 messages covering each intent type
  • Add a route_to_handler(intent: ClassifiedIntent) -> str that returns the handler name

Exercise 02

LangGraph State Machine with HITL

Build a 4-node LangGraph graph with human-in-the-loop interrupt before the execute node.

Objective

Implement a LangGraph workflow: classify → route → execute → respond, with an interrupt before execute so a human can approve or reject the action.

  • Define CopilotState TypedDict with: message, intent, action_plan, approved, response
  • Create 4 node functions: classify_node, route_node, execute_node, respond_node
  • Build the StateGraph, add nodes and edges, compile with interrupt_before=["execute_node"]
  • Simulate the HITL flow: run to interrupt, print the action plan, resume with human approval
  • Handle the rejection case: if approved=False, skip execute and go to respond with "Action cancelled"
copilot_graph.py

(y/n): ") # Resume with approval decision final = graph.invoke({"approved": approval == "y"}, config) print("Response:", final["response"])

Tool Approval Gate Middleware

Build a decorator-based approval gate that pauses tool execution for high-risk actions and requests human confirmation.

Objective

Create an @requires_approval decorator that intercepts tool calls, shows the user what the tool will do, and only proceeds if approved.

  • Implement @requires_approval(risk_level="high") decorator
  • The decorator should print the tool name and arguments before calling
  • For "high" risk: require explicit user confirmation ("yes"/"no" prompt)
  • For "medium" risk: log the action and proceed automatically
  • Raise ActionRejectedError if user denies, which the caller can catch
  • Apply the decorator to send_email, delete_file, and read_doc tools with appropriate risk levels

Full Copilot FastAPI Endpoint

Objective

Build a FastAPI endpoint that accepts a user message + user_id, runs through the full copilot pipeline, and returns a structured response with metadata.

  • POST /copilot endpoint accepting {"message": str, "user_id": str, "thread_id": str}
  • Inside the handler: classify intent → assemble context → invoke LangGraph → return response
