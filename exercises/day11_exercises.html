Disk-Based Embedding Cache

Build a persistent cache using SQLite that stores embeddings keyed by text hash, eliminating redundant API calls.

Completed
Objective
Goal: In production, the same document is often re-embedded repeatedly (e.g. on server restart). Build an EmbeddingCache that hashes each text with SHA-256, checks a SQLite database before calling the API, and persists results to disk across restarts.
Your Task

Build a persistent embedding cache:

  • Implement EmbeddingCache(db_path, model) — creates SQLite table on init
  • get_or_create(text: str) -> np.ndarray — checks cache first, calls API only on miss

  • Embed the same 10 texts twice — verify the second run makes 0 API calls (all cached)

Async Embedding Pipeline

Embed large corpora 5x faster using async/await with the OpenAI async client and asyncio.gather.

Completed
Objective
Goal: Use openai.AsyncOpenAI with asyncio.gather() to fire multiple embedding requests concurrently. Compare wall-clock time against the sequential BatchEmbedder from Exercise 2, demonstrating the I/O-bound speedup.
Your Task

Build an async batch embedder:

  • Use openai.AsyncOpenAI() and async def embed_batch(...)
  • Create async def embed_all(texts, batch_size) that gathers all batch tasks concurrently
  • Use asyncio.Semaphore(5) to limit to 5 concurrent requests (avoid rate limits)
  • Embed 200 texts and compare time vs sequential approach
  • Assert the output matrix shape is correct and all embeddings are non-zero
