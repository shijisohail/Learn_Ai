Production Dockerfile for FastAPI + Model

Write a multi-stage Dockerfile that produces a minimal, secure image for a FastAPI AI service using uv and a non-root user.

Completed
Multi-Stage Build — Builder → Runtime
Stage 1: builder
python:3.12-slim + uv
install deps
COPY --from=builder
Stage 2: runtime
python:3.12-slim
no build tools
non-root user
Size comparison: Naive single-stage image: ~1.2GB. Multi-stage with uv: ~180MB. That's 85% smaller — faster pulls, smaller attack surface, cheaper ECR/GCR storage.
Starter Dockerfile
# ── Stage 1: install dependencies ──────────────────────────── FROM python:3.12-slim AS builder # Install uv (fast Python package manager) RUN pip install uv WORKDIR /build # Copy dependency files first (layer caching) COPY pyproject.toml uv.lock ./ # TODO: install deps into /build/venv using uv # uv sync --frozen --no-dev --python python3.12 # ── Stage 2: runtime image ─────────────────────────────────── FROM python:3.12-slim # Create non-root user RUN groupadd -r appuser && useradd -r -g appuser appuser WORKDIR /app # Copy venv from builder stage COPY --from=builder /build/.venv /app/.venv # TODO: copy application code # TODO: set ENV PATH, switch to appuser, expose port, set CMD
Your Task

Complete the Dockerfile:

  • In builder: run RUN uv sync --frozen --no-dev to install into .venv
  • In runtime: COPY app/ /app/app/ and set ENV PATH="/app/.venv/bin:$PATH"
  • Switch to non-root: USER appuser
  • Add HEALTHCHECK --interval=30s CMD curl -f http://localhost:8000/health || exit 1
  • Set CMD ["uvicorn", "app.main:app", "--host", "0.0.0.0", "--port", "8000", "--workers", "2"]
  • Build and verify: docker build -t ai-service . && docker run --rm ai-service
Test Cases
T1
docker build -t ai-service . → success
→ Exit code 0, image created
T2
docker image inspect ai-service --format "{{.Size}}"
→ less than 300MB
T3
docker run ai-service whoami
→ "appuser" (non-root)
T4
docker run -p 8000:8000 ai-service → curl localhost:8000/health
→ 200 {"status": "ok"}
Progressive Hints
uv creates the venv at .venv inside the working directory. So COPY --from=builder /build/.venv /app/.venv then set ENV PATH="/app/.venv/bin:$PATH" and you're done — no need to activate.
Copy pyproject.toml and uv.lock BEFORE copying your app code. This way, Docker only re-runs the expensive uv sync when your dependencies change, not every time you change a Python file.

GPU in K8s requirements: Node must have NVIDIA device plugin DaemonSet running. Request and limit must be equal for GPU resources. Use node selectors or tolerations to target GPU node pools. Set NVIDIA_VISIBLE_DEVICES=all env var.
Starter Kubernetes Manifest
apiVersion: apps/v1 kind: Deployment metadata: name: ai-inference namespace: ai-prod labels: app: ai-inference version: v1 spec: replicas: 2 selector: matchLabels: app: ai-inference template: metadata: labels: app: ai-inference spec: # TODO: add nodeSelector for GPU nodes containers: - name: inference image: myregistry/ai-service:latest ports: - containerPort: 8000 resources: requests: memory: "4Gi" cpu: "2" # TODO: add GPU request limits: memory: "8Gi" cpu: "4" # TODO: add GPU limit (must equal request) # TODO: add liveness/readiness probes # TODO: add envFrom secretRef for ANTHROPIC_API_KEY
Your Task

Complete the K8s deployment manifest:

  • Add GPU resource: nvidia.com/gpu: "1" to both requests and limits
  • Add nodeSelector: accelerator: nvidia-tesla-a100
  • Add liveness probe: httpGet: path: /health port: 8000, initialDelaySeconds: 30
  • Add readiness probe: httpGet: path: /ready port: 8000, initialDelaySeconds: 10
  • Add secret reference: envFrom: - secretRef: name: ai-secrets
  • Also write a HorizontalPodAutoscaler that scales 2–10 replicas based on CPU at 70%
Test Cases

Progressive Hints
The resource key uses a dot-separated domain format: nvidia.com/gpu (not nvidia-gpu). Both requests and limits must have the exact same value — K8s does not allow fractional GPU requests.

Your Task

Implement all three endpoints:

  • /health: return {"status": "ok", "version": "1.0"} — never check external deps
  • /ready: ping Qdrant GET /healthz and Redis PING; return 503 + details if either fails
