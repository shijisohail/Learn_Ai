Fallback Router with Automatic Retries

Build a resilient caller that falls back from expensive to cheap models on error or high latency.

Completed
Objective
Goal: Use LiteLLM's fallbacks parameter or build your own fallback logic. Primary: GPT-4o. Fallback 1: Claude Haiku. Fallback 2: GPT-4o-mini. Implement rate-limit detection and exponential backoff using tenacity.
Your Task

Build a resilient LLM caller:

  • Use litellm.completion(model=..., fallbacks=[...]) with a primary + 2 fallback models
  • Wrap calls with @retry from tenacity — retry up to 3 times with exponential backoff on RateLimitError
  • Simulate a rate limit error by patching the primary model call to raise litellm.RateLimitError
  • Verify the fallback model was used and log which model actually answered
  • Add a timeout parameter (5 seconds) and test that slow models trigger fallback

Completed
Objective
Goal: Use litellm.completion(stream=True) to stream tokens from both OpenAI and Anthropic, measuring time-to-first-token (TTFT) and total tokens per second. Display streamed output progressively in the terminal.
Your Task

Build a streaming benchmark:

  • Call litellm.completion(model=..., messages=..., stream=True) for 2 models
  • Iterate over chunk.choices[0].delta.content and print each token as it arrives

  • Count total tokens received and calculate tokens/second throughput
  • Print a summary: TTFT, total time, throughput for each model

Prompt Caching with Claude

Use Anthropic's prompt caching to slash costs on repeated long-context calls by up to 90%.

Completed
Objective
Goal: Use cache_control: {"type": "ephemeral"} on a large system prompt with the Anthropic SDK. Make the same call twice and observe that the second call is cheaper (cached input tokens vs uncached). Calculate actual savings.
Your Task

Implement and measure prompt caching:

  • Build a large system prompt (>1024 tokens) — load a text file or repeat content
  • Mark it with {"cache_control": {"type": "ephemeral"}} in the content block
  • Make the same call twice with the same system prompt but different user messages
  • Compare usage.cache_read_input_tokens vs usage.cache_creation_input_tokens
  • Calculate and print: cost without caching vs cost with caching, and % savings
