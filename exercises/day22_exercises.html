Tree of Thought Prompting

Implement a ToT prompt that generates multiple reasoning branches, evaluates each, and selects the best answer via a voting step.

Completed
Tree of Thought — How Branching Reasoning Works
Problem
→
Branch A
Approach 1
+
Branch B
Approach 2
+
Branch C
Approach 3
Evaluate each branch score (0–10)
→
Best branch → Final Answer
Why it works: LLMs solve hard problems better when forced to explore multiple solution paths and then self-evaluate. ToT mimics human brainstorming: generate diverse approaches, score them, commit to the best.

Self-Consistency Voting

Sample the same prompt multiple times at high temperature, extract final answers, and return the majority vote.

Completed
Self-Consistency — Majority Vote Over Samples
Same Prompt
→ × k
Sample 1 → ans A
Sample 2 → ans A
Sample 3 → ans B
Sample 4 → ans A
Sample 5 → ans A
→
Vote: A (4/5) ✓
Key insight: A single sample from an LLM may land on an incorrect reasoning path. By sampling k=5–10 times and taking the majority answer, accuracy on multi-step math and logic tasks increases by 10–20% without fine-tuning.

HyDE — Hypothetical Document Embeddings

Generate a fake "ideal document" for a query, embed it, then use that embedding for vector search instead of the raw query.

Completed
HyDE Pipeline — Query Expansion via Hallucination
Short Query
"vector DB latency"
→ LLM
Hypothetical Doc
200-word "ideal" passage
→ embed
Dense Vector
→ ANN search
Real Documents
Why HyDE works: Short queries embed into a sparse region of vector space. A hypothetical document that answers the query embeds into the same dense region as real documents that answer it — giving dramatically better retrieval recall.

Anthropic Prompt Caching

Use cache_control breakpoints to cache a large system prompt, then measure the token savings on subsequent calls.

Completed
Prompt Caching — Cache the Static Prefix
Call 1: Full tokens billed
system=5000 tokens + cache write
Call 2: Cache HIT
system=90% cheaper + dynamic part only
Call 3: Cache HIT
system=90% cheaper again
Cache lifetime: 5 min
Anthropic
Cache Store
Cost math: Cache writes cost 25% extra. Cache reads cost 10% of normal. On a 5000-token system prompt with 100 calls: savings = 99 × 5000 × 0.9 × $3/1M = ~$1.34 saved in one session.

Progressive Hints
The system block must be a list: [{"type": "text", "text": KNOWLEDGE_BASE, "cache_control": {"type": "ephemeral"}}]. The cache breakpoint applies at the end of this text block.
Anthropic only caches prompts over ~1024 tokens. If your KNOWLEDGE_BASE is too short, caching won't activate. Repeat the string or use a real document to hit the threshold.
