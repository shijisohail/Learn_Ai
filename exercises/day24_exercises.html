Retry Loop on Validation Failure

Build a retry wrapper that catches ValidationError, appends the error to the conversation, and re-asks the model to fix its output.

Error-in-prompt technique: Append the exact Pydantic ValidationError message to the conversation as a "user" correction. The model sees its mistake and the schema constraint, making self-correction highly reliable (95%+ on first retry).

# pip install openai pydantic import json from openai import OpenAI from pydantic import BaseModel, ValidationError from typing import TypeVar, Type T = TypeVar("T", bound=BaseModel) client = OpenAI() def extract_with_retry( prompt: str, model_class: Type[T], max_retries: int = 3 ) -> T: """Call LLM, parse JSON, validate. Retry on failure with error feedback.""" messages = [ {"role":"system","content":f"Output valid JSON matching: {model_class.model_json_schema()}"}, {"role":"user","content":prompt} ] for attempt in range(max_retries): resp = client.chat.completions.create( model="gpt-4o-mini", response_format={"type":"json_object"}, messages=messages ) raw = resp.choices[0].message.content try: # TODO: parse and validate, return on success ... except (json.JSONDecodeError, ValidationError) as e: # TODO: append assistant response + error correction to messages, continue loop ... raise RuntimeError(f"Failed after {max_retries} attempts")
Your Task

Complete the retry loop:

  • On success: parse JSON, call model_class.model_validate(data), return immediately
  • On failure: append the raw assistant reply as a "assistant" message, then append a "user" message containing the validation error string
  • Track attempt count and log each retry with the error message
  • Test by intentionally passing a prompt that causes the model to omit a required field on first attempt
Test Cases
T1
result = extract_with_retry("List of 3 Python books", BookList, max_retries=3)
→ isinstance(result, BookList)
T2
extract_with_retry("bad prompt", ModelWithRequiredField, max_retries=1)
→ raises RuntimeError after 1 attempt
T3
Messages list grows by 2 per retry (assistant + user correction)
→ len(messages) == 2 + 2*retries_used
Progressive Hints
messages.append({"role": "assistant", "content": raw}) then messages.append({"role": "user", "content": f"Your response failed validation:\n{str(e)}\n\nPlease fix and return valid JSON."})

  • In extract_paper(): pass max_retries=Retrying(stop=stop_after_attempt(3), wait=wait_exponential(multiplier=1, min=1, max=10))
