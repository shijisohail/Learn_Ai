Secure the copilot endpoint with JWT authentication, isolate each tenant's data in Qdrant using payload filters, and restrict tools based on user roles.

MLflow Experiment Comparison

Run 3 different RAG configurations as MLflow experiments, compare metrics, and promote the best to staging.

Capstone Layer 5 — MLOps

Use MLflow to systematically compare RAG configurations and promote the winner to the model registry.

  • Define 3 RAG configs: (1) chunk_size=256, no rerank; (2) chunk_size=512, with rerank; (3) chunk_size=512, HyDE enabled
  • For each config, simulate a test run logging: faithfulness, answer_relevancy, avg_latency_ms, chunk_size to MLflow
  • Use mlflow.search_runs to find the run with highest faithfulness score
  • Register the best run as a model version using mlflow.register_model
  • Transition the registered model to "Staging" using MlflowClient.transition_model_version_stage
