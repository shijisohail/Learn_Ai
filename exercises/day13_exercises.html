Exercise 1 of 5

Your First CrewAI Crew

Define agents, tasks, and a crew in Python and run a simple sequential content creation pipeline.

Completed
Objective
Goal: Create a 2-agent crew: a Researcher who collects facts about a topic, and a Writer who turns those facts into a polished blog post. Use CrewAI's Agent, Task, and Crew classes with sequential execution. The Writer's task should receive the Researcher's output via context.
Your Task

Build a content creation crew:

  • Define a researcher Agent with role, goal, and backstory — focused on Python async programming
  • Define a writer Agent with role, goal, and backstory — focused on clear technical writing
  • Create a research_task that asks the researcher to gather 5 key facts about asyncio
  • Create a writing_task with context=[research_task] — the writer uses the research output
  • Instantiate a Crew with both agents and tasks, run it, and print the final output
Starter Code
ex1_first_crew.pyPython 3.11+
# pip install crewai langchain-openai from crewai import Agent, Task, Crew, Process from langchain_openai import ChatOpenAI llm = ChatOpenAI(model="gpt-4o-mini", temperature=0.3) # TODO: define the researcher agent researcher = Agent( role="Python Async Expert", goal="Research and compile accurate technical facts about Python asyncio", backstory="""You are a senior Python engineer with 10 years of experience in async programming. You specialise in asyncio, aiohttp, and concurrent patterns.""", llm=llm, verbose=True, allow_delegation=False, ) # TODO: define the writer agent writer = Agent( role="Technical Writer", goal="Transform technical research into clear, engaging blog posts", backstory="""You are a developer advocate who writes technical content for software engineers. Your posts are practical, accurate, and easy to follow.""", llm=llm, verbose=True, allow_delegation=False, ) # TODO: define research_task research_task = Task( description="""Research Python asyncio and compile exactly 5 key facts. Focus on: event loops, coroutines, gather(), tasks, and common pitfalls. Each fact should be 1-2 sentences with a practical insight.""", expected_output="A numbered list of 5 technical facts about Python asyncio", agent=researcher, ) # TODO: define writing_task with context=[research_task] writing_task = Task( description="""Using the research provided, write a 300-word blog post titled 'Mastering Python asyncio: 5 Things Every Developer Should Know'. Include an intro, one paragraph per fact, and a brief conclusion.""", expected_output="A complete 300-word blog post in markdown format", agent=writer, context=[research_task], # receives research_task output ) # TODO: create and run the crew crew = Crew( agents=[researcher, writer], tasks=[research_task, writing_task], process=Process.sequential, verbose=True, ) result = crew.kickoff() print("\n" + "="*60) print(result)

Hints
When context=[research_task] is set on the writing task, CrewAI automatically injects the research task's output into the writer's context. The writer sees it as additional information before its own task description.
The expected_output field is shown to the agent and used by CrewAI to validate the output format. Be specific — "A numbered list of 5 facts" is better than "some facts". CrewAI will retry if the output doesn't match.

Equipping Agents with Custom Tools

Build custom tools using @tool decorator and give specific agents access to them.

Completed
Objective
Goal: Use CrewAI's @tool decorator to define two custom tools: a fetch_python_docs tool (mocked) and a run_code_snippet tool (using subprocess). Assign tools to specific agents and verify they get called during execution.
Your Task

Add tools to a code review crew:

  • Implement @tool fetch_python_docs(topic: str) -> str — returns a mock doc string for given Python topic
  • Implement @tool run_code_snippet(code: str) -> str — runs Python code via subprocess and returns stdout/stderr
  • Create a Code Analyst agent with both tools and task it to analyse a code snippet
  • Create a Code Reviewer agent with only the docs tool, tasked with writing review comments
  • Verify the Code Analyst actually calls run_code_snippet on the provided code
Starter Code
ex2_tools.pyPython 3.11+
# pip install crewai langchain-openai from crewai import Agent, Task, Crew, Process from crewai.tools import tool from langchain_openai import ChatOpenAI import subprocess import textwrap @tool def fetch_python_docs(topic: str) -> str: """Fetch Python documentation for a given topic (async, generators, decorators, etc).""" MOCK_DOCS = { "async": "asyncio provides infrastructure for writing single-threaded concurrent code using coroutines.", "generators": "Generators use yield to produce values lazily. They implement the iterator protocol.", "decorators": "Decorators are functions that modify other functions using @syntax.", "context managers": "Context managers implement __enter__ and __exit__ for resource management.", } key = topic.lower() return MOCK_DOCS.get(key, f"No documentation found for '{topic}'. Try: async, generators, decorators.") @tool def run_code_snippet(code: str) -> str: """Execute a Python code snippet and return stdout/stderr. Max 5 seconds.""" try: # TODO: use subprocess to run the code safely with timeout result = subprocess.run( ["python", "-c", textwrap.dedent(code)], capture_output=True, text=True, timeout=5 ) output = result.stdout or "(no output)" error = result.stderr return f"STDOUT:\n{output}\nSTDERR:\n{error}" if error else f"STDOUT:\n{output}" except subprocess.TimeoutExpired: return "ERROR: Code timed out after 5 seconds" except Exception as e: return f"ERROR: {e}" CODE_TO_REVIEW = """ def fibonacci(n): if n <= 1: return n return fibonacci(n-1) + fibonacci(n-2) print(fibonacci(10)) print(fibonacci(20)) """ llm = ChatOpenAI(model="gpt-4o-mini") # TODO: analyst agent with both tools analyst = Agent( role="Code Analyst", goal="Run code and identify performance issues", backstory="Expert in Python performance profiling and code analysis.", tools=[run_code_snippet, fetch_python_docs], llm=llm, verbose=True, ) # TODO: reviewer agent with docs tool only reviewer = Agent( role="Code Reviewer", goal="Write actionable code review comments with improvement suggestions", backstory="Senior engineer who writes clear, constructive code review feedback.", tools=[fetch_python_docs], llm=llm, verbose=True, ) analysis_task = Task( description=f"Run this code and identify performance issues:\n{CODE_TO_REVIEW}", expected_output="Analysis report: runtime output, identified issues, complexity assessment", agent=analyst, ) review_task = Task( description="Based on the analysis, write 3 specific improvement suggestions with code examples.", expected_output="3 numbered review comments with improved code snippets", agent=reviewer, context=[analysis_task], ) crew = Crew(agents=[analyst, reviewer], tasks=[analysis_task, review_task], verbose=True) print(crew.kickoff())

Exercise 3 of 5

YAML-Configured Crew

Define agents and tasks entirely in YAML config files — the production-ready CrewAI pattern.

Completed
Objective
Goal: Use CrewAI's YAML configuration pattern — define agents in agents.yaml and tasks in tasks.yaml, then load them in Python with @CrewBase decorator. This is the recommended pattern for production crews where non-engineers can tweak prompts without touching Python.
Your Task

Build a YAML-configured documentation crew:

  • Create config/agents.yaml with a docs_writer and editor agent configuration
  • Create config/tasks.yaml with a write_docs and edit_docs task configuration
  • Create a Python crew class using @CrewBase, @agent, @task, and @crew decorators
  • Run the crew with crew_instance.crew().kickoff(inputs={"topic": "Python type hints"})
  • Edit the YAML to change the writer's tone — verify the output changes without touching Python
Starter Code
config/agents.yamlYAML
# config/agents.yaml docs_writer: role: > Technical Documentation Writer goal: > Write comprehensive, accurate documentation for {topic} backstory: > You are a senior technical writer who creates clear Python documentation. Your writing is concise, example-driven, and follows Google style guide. editor: role: > Technical Editor goal: > Review and improve documentation for clarity, accuracy, and completeness backstory: > You are a meticulous editor with a background in Python development. You catch errors, improve examples, and ensure docs match best practices.
config/tasks.yamlYAML
# config/tasks.yaml write_docs: description: > Write a comprehensive documentation section for {topic}. Include: overview, key concepts, 2 code examples, and common pitfalls. Target audience: Python developers with 1-3 years of experience. expected_output: > A 400-word documentation section in markdown with 2 runnable code examples agent: docs_writer edit_docs: description: > Review the documentation draft and improve it. Check: technical accuracy, example correctness, clarity, and completeness. Produce the final polished version. expected_output: > Final polished documentation with editor comments explaining changes made agent: editor
docs_crew.pyPython 3.11+
from crewai import Agent, Crew, Process, Task from crewai.project import CrewBase, agent, crew, task from langchain_openai import ChatOpenAI @CrewBase class DocsCrew: """Documentation writing crew.""" agents_config = "config/agents.yaml" tasks_config = "config/tasks.yaml" def _llm(self): return ChatOpenAI(model="gpt-4o-mini") # TODO: define docs_writer agent using @agent decorator @agent def docs_writer(self) -> Agent: return Agent( config=self.agents_config["docs_writer"], llm=self._llm(), verbose=True, ) # TODO: define editor agent @agent def editor(self) -> Agent: return Agent( config=self.agents_config["editor"], llm=self._llm(), verbose=True, ) # TODO: define tasks using @task decorator @task def write_docs(self) -> Task: return Task(config=self.tasks_config["write_docs"]) @task def edit_docs(self) -> Task: return Task(config=self.tasks_config["edit_docs"]) # TODO: define the crew using @crew decorator @crew def crew(self) -> Crew: return Crew( agents=self.agents, tasks=self.tasks, process=Process.sequential, verbose=True, ) if __name__ == "__main__": result = DocsCrew().crew().kickoff(inputs={"topic": "Python type hints"}) print(result)

Completed
Objective
Goal: Switch from Process.sequential to Process.hierarchical. A Project Manager agent (using a stronger model) autonomously decides which specialists to call and in what order — you don't pre-define task dependencies. Observe how the manager delegates to specialists.
Your Task

  • Create 3 specialist agents: API Designer, Security Reviewer, Performance Analyst

Exercise 5 of 5

Multi-Step Research Pipeline with Output Dependencies

Build a 4-agent pipeline where each agent's output feeds into the next via explicit context dependencies.

Completed
Objective
Goal: Build a software evaluation pipeline: Scout (identifies Python libraries for a use case) → Analyst (evaluates each library's pros/cons) → Comparator (creates a comparison table) → Recommender (makes a final recommendation). Each task depends on the previous via context.
Your Task

Build a 4-stage library evaluation pipeline:

  • Scout agent: given a use case, identify 3 relevant Python libraries
  • Analyst agent: for each library, evaluate: maturity, performance, ecosystem, ease of use
  • Comparator agent: create a markdown comparison table from the analysis
  • Recommender agent: make a final recommendation with justification based on the comparison
  • Test with use case: "vector database for a Python RAG application"
Starter Code
ex5_pipeline.pyPython 3.11+
from crewai import Agent, Task, Crew, Process from langchain_openai import ChatOpenAI llm = ChatOpenAI(model="gpt-4o-mini") # TODO: define 4 agents scout = Agent(role="Library Scout", goal="Identify the most relevant Python libraries for a given use case", backstory="Python ecosystem expert who keeps up with the latest libraries and trends.", llm=llm, verbose=True, allow_delegation=False) analyst = Agent(role="Library Analyst", goal="Evaluate Python libraries on maturity, performance, ecosystem, and usability", backstory="Technical evaluator who benchmarks and assesses Python packages for production use.", llm=llm, verbose=True, allow_delegation=False) comparator = Agent(role="Comparison Specialist", goal="Create clear comparison tables from technical evaluations", backstory="Technical writer who specialises in structured comparisons and decision frameworks.", llm=llm, verbose=True, allow_delegation=False) recommender = Agent(role="Strategic Recommender", goal="Make clear, justified technology recommendations based on evidence", backstory="CTO advisor who synthesises technical analysis into actionable decisions.", llm=llm, verbose=True, allow_delegation=False) # TODO: define 4 tasks with context dependencies scout_task = Task( description="Identify the top 3 Python libraries for: vector database for a Python RAG application. For each, provide name, GitHub link, and 1-sentence description.", expected_output="A list of exactly 3 libraries with name, URL, and description", agent=scout, ) analysis_task = Task( description="For each library identified by the scout, evaluate: maturity (1-10), performance (1-10), ecosystem (1-10), ease of use (1-10). Include reasoning for each score.", expected_output="Detailed evaluation of all 3 libraries with numerical scores and reasoning", agent=analyst, context=[scout_task], ) # TODO: comparison_task with context=[analysis_task] comparison_task = Task( description="Create a markdown comparison table from the analysis. Columns: Library | Maturity | Performance | Ecosystem | Ease of Use | Overall", expected_output="A markdown table comparing all 3 libraries with a total score row", agent=comparator, context=[analysis_task], ) # TODO: recommendation_task with context=[comparison_task] recommendation_task = Task( description="Based on the comparison table, recommend one library for a Python RAG application. Provide: recommendation, 3 key reasons, migration considerations, and one risk to watch.", expected_output="Final recommendation with justification, reasons, and risk assessment", agent=recommender, context=[comparison_task], ) crew = Crew( agents=[scout, analyst, comparator, recommender], tasks=[scout_task, analysis_task, comparison_task, recommendation_task], process=Process.sequential, verbose=True, ) result = crew.kickoff() print("\n===== FINAL RECOMMENDATION =====") print(result)

Hints
A task can have context=[task1, task2, task3] — it receives the outputs of all listed tasks. This lets you fan-in multiple branches. For a linear pipeline, each task only needs the immediately preceding task in its context.
