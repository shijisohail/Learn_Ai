Visual Explainer — Why LangGraph?

While Loop (before)

while True: response = llm(messages) if response.stop == "end": break # execute tools... messages.append(tool_result)
✗ No explicit state schema
✗ Hard to add checkpointing
✗ Can't pause / resume
✗ Routing logic buried in code
✗ Hard to debug / visualize

LangGraph (now)

START → [agent] → [tools] ↑ ↓ └─────────┘ (loop until done)
✓ Typed state (TypedDict)
✓ Built-in checkpointing (SQLite)
✓ Interrupt / resume at any node
✓ Explicit routing functions
✓ Inspectable graph structure

State Design Lab

▼
Visual Explainer — TypedDict Schema Anatomy

AgentState field-by-field — hover to see details

AgentState (TypedDict)
├── messages: Annotated[list, add_messages] ← APPENDS — never replaces
└── [HumanMsg, AIMsg, ToolMsg, AIMsg ...] grows with every node
├── query: str ← REPLACES — last write wins
├── search_results: list[str] ← REPLACES — set by researcher node
├── draft: str ← REPLACES — set by writer node
└── is_complete: bool ← REPLACES — True = graph exits

add_messages vs plain replace — what happens when two nodes write simultaneously

messages: add_messages (MERGES)
Node A writes: [AIMessage("Hello")]
Node B writes: [ToolMessage("result")]
Result: [AIMessage("Hello"), ToolMessage("result")] ✓
draft: str (REPLACES — last wins)
Node A writes: "Draft version 1..."
Node B writes: "Draft version 2..." ← wins
Result: "Draft version 2..." (v1 is gone)

State flows through nodes — merge semantics

State IN: {messages: [HumanMsg], query: "FastAPI docs", draft: "", search_results: []} │ ▼ research_node(state) returns only: {search_results: ["result1", "result2"]} │ LangGraph merges → unchanged fields preserved │ State OUT: {messages: [HumanMsg], query: "FastAPI docs", draft: "", search_results: ["result1", "result2"]} │ ▼ writer_node(state) returns only: {draft: "Based on results...", is_complete: True} │ State OUT: {messages: [HumanMsg], query: "FastAPI docs", draft: "Based on results...", ...}

Your Task — Build a 4-Way Router

  • router(state) → "tools" | "error_handler" | "clarification" | END
  • Tool call detected → "tools"
  • Agent is done (end_turn, no tool calls) → END

Returns: "tools" | "error_handler" | "clarification" | END """ messages = state["messages"] if not messages: return END last = messages[-1] # Case 1: Tool call requested if hasattr(last, "tool_calls") and last.tool_calls: return "tools" # Case 2: Error in response # TODO: check last.content for "ERROR" keyword ... # Case 3: Needs clarification # TODO: check if content ends with "?" ... # Case 4: Default — we're done return END # Wire it up in graph: graph.add_conditional_edges( "agent", router, { "tools": "tools", "error_handler": "error_handler", "clarification": "clarification", END: END, } )

from typing import Annotated, TypedDict, Optional from langgraph.graph import StateGraph, START, END from langgraph.graph.message import add_messages from langgraph.checkpoint.sqlite import SqliteSaver from langgraph.types import Command, interrupt import uuid, sqlite3 import anthropic # ── STATE ────────────────────────────────────────────────────────── class ResearchState(TypedDict): messages: Annotated[list, add_messages] query: str search_results: list[str] human_feedback: Optional[str] # set after interrupt draft: str is_approved: bool iteration: int # retry count after rejections # ── NODES ────────────────────────────────────────────────────────── client = anthropic.Anthropic() def research_node(state: ResearchState) -> dict: """Call web search (mock), store results.""" query = state["query"] feedback = state.get("human_feedback", "") # Mock search — replace with real search tool results = [ f"Result 1: {query} — performance improved 40% in latest release", f"Result 2: {query} — new features include native WebSocket support", f"Result 3: {query} — documentation updated with async examples", ] if feedback: results.append(f"Focused result per feedback: {feedback}") return {"search_results": results, "iteration": state.get("iteration", 0) + 1} def review_node(state: ResearchState) -> Command: """Pause for human review — returns approved/rejected + optional feedback.""" # interrupt() saves state to SQLite and raises an exception # app.invoke() returns the state at this point # resume with: app.invoke(Command(resume={"approved": True, "feedback": "..."}), config=...) human_input = interrupt({ "results": state["search_results"], "question": "Approve these research results?" }) if human_input["approved"]: return Command(goto="writer", update={ "is_approved": True, "human_feedback": human_input.get("feedback", "") }) else: # Reject → go back to research with feedback return Command(goto="research", update={ "is_approved": False, "human_feedback": human_input.get("feedback", "Improve coverage") }) def writer_node(state: ResearchState) -> dict: """Write final report using approved research.""" context = "\n".join(state["search_results"]) feedback = state.get("human_feedback", "") prompt = f"""Write a concise research report on: {state['query']} Research findings: {context} {f'Human feedback: {feedback}' if feedback else ''} Write a 3-paragraph report.""" response = client.messages.create( model="claude-opus-4-5", max_tokens=512, messages=[{"role": "user", "content": prompt}] ) return {"draft": response.content[0].text} # ── GRAPH ────────────────────────────────────────────────────────── def build_hitl_graph(db_path: str = ":memory:"): conn = sqlite3.connect(db_path, check_same_thread=False) checkpointer = SqliteSaver(conn) graph = StateGraph(ResearchState) graph.add_node("research", research_node) graph.add_node("review", review_node) graph.add_node("writer", writer_node) graph.add_edge(START, "research") graph.add_edge("research", "review") # review_node uses Command() to route dynamically graph.add_edge("writer", END) return graph.compile(checkpointer=checkpointer, interrupt_before=["review"]) # ── USAGE ────────────────────────────────────────────────────────── app = build_hitl_graph("research_checkpoints.db") thread_id = str(uuid.uuid4()) config = {"configurable": {"thread_id": thread_id}} # Phase 1: Run until interrupt state = app.invoke({ "query": "FastAPI performance improvements", "messages": [], "search_results": [], "human_feedback": None, "draft": "", "is_approved": False, "iteration": 0 }, config=config) print("PAUSED — review required. Results:", state["search_results"]) # Phase 2: Human approves — resume same thread final = app.invoke( Command(resume={"approved": True, "feedback": "Great results"}), config=config # same thread_id! ) print("COMPLETE. Draft:", final["draft"])

  • interrupt(): from langgraph.types import interrupt — call inside a node, graph pauses and returns state to caller
  • Command(resume=...): from langgraph.types import Command — pass as first arg to app.invoke() to resume
  • SQLiteSaver: pip install langgraph-checkpoint-sqlite — use :memory: for tests, file path for production
  • interrupt_before vs interrupt_after: interrupt_before=["review"] pauses BEFORE review_node runs — state is saved with research results
  • Inspect checkpoint: app.get_state(config) — see full state at any point, including all past checkpoints
